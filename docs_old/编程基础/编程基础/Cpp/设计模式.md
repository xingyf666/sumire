# 设计模式

在实际编程中，我们常常需要对整个程序框架进行设计，以便于开发和维护代码。下面将列举一些常见的设计技巧和思想。



## 基本原则

1. **开放封闭**：类的改动通过增加代码实现，而不是修改源码
2. 单一职责：类的职责要单一，对外只提供一种功能，引起类变化的原因应该只有一个
3. **依赖倒置**：依赖于抽象，不要依赖具体的实现，也就是针对接口编程
4. 接口隔离：不应该强迫用户的程序依赖他们不需要的接口方法。一个接口应该只提供一种对外功能，不应该把所有操作都封装到一个接口中
5. 里式替换：任何抽象类出现的地方都可以用它的实现类进行替换
6. 优先使用组合而不是继承
7. **迪米特法则**：一个对象应当对其它对象尽可能少地了解，从而降低各个对象之间的耦合，提高系统的可维护性



### 开放封闭原则

例如我们定义一个收营员的类

```c++
class BankWorker
{
public:
    void move()
    {
        cout << "Moving Bank" << endl;
    }
    void save()
    {
        cout << "Saving Bank" << endl;
    }
    void wage()
    {
        cout << "Get wage" << endl;
    }
};
```

这种写法的问题在于如果要增加类的功能，就需要修改类的成员。如果我们使用抽象类就可以避免这一问题

```c++
class AbBankWorker
{
public:
    virtual void dothing() = 0;
};

class MoveBankWorker : public AbBankWorker
{
public:
    void dothing()
    {
        cout << "Moving Bank" << endl;
    }
};

class SaveBankWorker : public AbBankWorker
{
public:
    void dothing()
    {
        cout << "Saving Bank" << endl;
    }
};

class WageBankWorker : public AbBankWorker
{
public:
    void dothing()
    {
        cout << "Get wage" << endl;
    }
};
```

要增加功能，只需要定义新的类继承抽象类即可。通过定义一个接口函数可以便捷地使用

```c++
// 框架函数
void Todo(AbBankWorker* worker)
{
    worker->dothing();
}
```

利用多态实现封装代码。



### 依赖倒置原则

好的设计模式应该是业务逻辑和实现都向抽象层依赖，才能实现解耦合

![](image-20220717113712849.png)



我们设计一个 Computer 框架和具体组件的解耦合

```c++
class HardDisk
{
public:
    virtual void work() = 0;
};

class Memory
{
public:
    virtual void work() = 0;
};

class Cpu
{
public:
    virtual void work() = 0;
};

class InterCpu : public Cpu
{
public:
    void work() override
    {
        cout << "InterCpu" << endl;
    }
};

class Computor
{
public:
    Computor(HardDisk *hardDisk, Memory *memory, Cpu *cpu)
    {
        m_harddisk = hardDisk;
        m_memory = memory;
        m_cpu = cpu;
    }

public:
    void work()
    {
        m_harddisk->work();
        m_memory->work();
        m_cpu->work();
    }

private:
    HardDisk *m_harddisk;
    Memory *m_memory;
    Cpu *m_cpu;
};
```

我们通过将其它类注入到一个类当中来实现增强功能，一般推荐使用这种组合方法扩充类的功能而不是使用继承。



## 创建型模式

### 单例模式

有些时候，我们希望全局仅且仅能创建唯一一个对象

* 构造函数私有化
* 定义唯一指向唯一对象的私有静态本地变量
* 提供一个公共接口访问单例对象
* 赋值私有化，禁止赋值



#### 懒汉模式

在调用返回函数时才进行构造

```cpp
class Rocket
{
private:
    Rocket() {}              // 私有构造函数，在外部无法创建该对象
    ~Rocket() {}             // 私有析构函数，防止在外部 delete 对象指针出现问题
    static Rocket *m_rocket; // 私有静态本地变量，定义对象指针而非对象是为了更加灵活，随时定义和删除

    void operator=(const Rocket &rocket) {} // 私有化赋值操作

public:
    // 静态本地函数，从而能通过类调用返回 Rocket
    static Rocket *shareRocket()
    {
        // 这里要考虑多线程同时调用的问题
        if (m_rocket == NULL)
        {
            m_rocket = new Rocket();
        }
        return m_rocket;
    }

    // 清空对象
    static Rocket *deleteRocket()
    {
        // 这里要考虑多线程同时调用的问题
        if (m_rocket != NULL)
        {
            delete m_rocket;
            m_rocket = NULL;
        }
        return m_rocket;
    }
};
```

我们通过如下方式实现单例对象的定义

```cpp
Rocket *Rocket::m_rocket = NULL;    // 外部初始化为空指针
Rocket *p = Rocket::shareRocket();  // 产生唯一的对象
Rocket *p1 = Rocket::shareRocket(); // 获得的仍然是唯一的对象
```



#### 饿汉模式

懒汉模式在使用时才会构造对象指针，这会导致一个缺点：在处理多线程问题时，可能出现多次创建的对象。因此我们可以调整程序在初始化时立即构造指针

```c++
class Rocket
{
private:
    Rocket() {}              // 私有构造函数，在外部无法创建该对象
    ~Rocket() {}             // 私有析构函数，防止在外部 delete 对象指针出现问题
    static Rocket *m_rocket; // 私有静态本地变量，定义对象指针而非对象是为了更加灵活，随时定义和删除

    void operator=(const Rocket &rocket) {} // 私有化赋值操作

public:
    // 静态本地函数，从而能通过类调用返回 Rocket
    static Rocket *shareRocket()
    {
        return m_rocket;
    }

    // 清空对象
    static Rocket *deleteRocket()
    {
        if (m_rocket != NULL)
        {
            delete m_rocket;
            m_rocket = NULL;
        }
        return m_rocket;
    }
};
```

然后在外部初始化直接构造

```c++
Rocket *Rocket::m_rocket = new Rocket; // 直接外部初始化
```



#### 多线程加锁

对多线程情况做互斥锁，从而避免多线程导致多次初始化的问题

```cpp
class singleton
{
  private:
    singleton()
    {
    }

    static singleton *p;
    static std::mutex lock_;

  public:
    static singleton *instance();
};

singleton *singleton::p = nullptr;

singleton *singleton::instance()
{
    std::lock_guard<std::mutex> guard(lock_);
    if (p == nullptr)
        p = new singleton();
    return p;
}
```

不过这种写法非常低效，多线程访问会被阻塞，导致效率降低。



#### 双重检查锁模式

通过执行两次检查来提高效率：由于大多数情况下 p 非空，因此就省去了上锁操作；第二次检查是为了防止上锁之前的访问顺序错误，导致 p 非空时也进入内部的问题。

```cpp
class singleton
{
  private:
    singleton()
    {
    }

    static singleton *p;
    static std::mutex lock_;

  public:
    // 双重检查
    static singleton *instance()
    {
        // 只有为空时才上锁
        if (p == nullptr)
        {
            std::lock_guard<std::mutex> guard(lock_);
            if (p == nullptr)
                p = new singleton();
        }
        return p;
    }

    // 实现一个内嵌垃圾回收类
    class CGarbo
    {
      public:
        ~CGarbo()
        {
            if (singleton::p)
                delete singleton::p;
        }
    };

    // 定义一个静态成员变量，程序结束时，系统会自动调用它的析构函数从而释放单例对象指针
    static CGarbo Garbo;
};

singleton *singleton::p = nullptr;
singleton::CGarbo Garbo;
std::mutex singleton::lock_;
```



在 C++11 中将

1. 分配内存
2. 完成初始化
3. 指向内存

这一流程视为原子操作，因此不会出现执行顺序重排的问题。但是在 C++11 之前存在问题，关键在于

```cpp
p = new singleton();
```

由于多线程的执行顺序问题，可能出现如下情况：

1. A 线程进入内部，首先为 p 分配了内存，但是**还未调用构造函数初始化**；
2. B 线程判断 p 已经分配内存，于是没有进入内部上锁，直接返回 p；

现在用户将得到一个分配了内存但是没有初始化的变量，可能导致错误。



#### 静态局部变量

一种更简单的写法是利用静态局部变量

```cpp
singleton &singleton::instance()
{
    static singleton p;
    return p;
}
```

由于 C++11 提出了内存模型，规定了**当一个线程正在初始化一个变量的时候，其他线程必须得等到该初始化完成以后才能访问它**。因此从 C++11 开始，这种写法是线程安全的，但对于之前的版本则不一定正确。



#### 原子操作

借助原子操作，可以规范原子量的读写。使用 `std::atomic_thread_fence` 添加内存模式标记

```cpp
std::mutex singleton::lock_;
std::atomic<singleton *> singleton::p;

singleton *singleton::instance()
{
    singleton *tmp = p.load(std::memory_order_relaxed);
    
    // 要求之后的所有读写操作不能排到前面
    std::atomic_thread_fence(std::memory_order_acquire);
    if (tmp == nullptr)
    {
        std::lock_guard<std::mutex> guard(lock_);
        tmp = p.load(std::memory_order_relaxed);
        if (tmp == nullptr)
        {
            tmp = new singleton();
            
            // 要求之前的所有读写操作不能排到后面
            std::atomic_thread_fence(std::memory_order_release);
            p.store(tmp, std::memory_order_relaxed);
        }
    }
    return p;
}
```



### 工厂模式

工厂是一个用于创建实例的类，外部直接调用工厂对象获取产品对象。

![](image-20220717152816775.png)

我们直接给出带有模板的抽象工厂模式：

```c++
#include <map>
#include <memory>
#include <stdexcept>
#include <string>

// 模板参数：对象类型，ID 类型，创建函数类型
template <class AbstractProduct, typename IdType, typename ProductCreator = std::unique_ptr<AbstractProduct> (*)()>
class GenericFactory
{
private:
    // 存放创建函数的 map
    using CallbackMap = std::map<IdType, ProductCreator>;

public:
    // 产生工厂实例，返回引用，程序结束自动析构
    static GenericFactory &createFactory()
    {
        static GenericFactory object;
        return object;
    }

    // 根据 id 注册创建函数，返回创建是否成功
    bool registerProduct(IdType Id, ProductCreator createFn)
    {
        return callbacks_.insert(typename CallbackMap::value_type(Id, createFn)).second;
    }

    // 销毁函数，返回是否存在该 id
    bool unregisterProduct(IdType Id)
    {
        return callbacks_.erase(Id) == 1;
    }

    // 根据 id 创建对象的智能指针
    template <class... TS>
    std::unique_ptr<AbstractProduct> createProduct(IdType Id, TS &&...args)
    {
        auto it = callbacks_.find(Id);
        if (it == callbacks_.end())
        {
            throw std::runtime_error("Unknown product ID.");
        }
        // 可变参数调用返回智能指针，使用 forward 转发参数
        return (it->second)(std::forward<TS>(args)...);
    }

private:
    // 构造全部私有化
    GenericFactory() = default;
    GenericFactory(const GenericFactory &) = default;
    GenericFactory &operator=(const GenericFactory &) = default;
    
    // 防止主动摧毁对象
    ~GenericFactory() = default;

private:
    CallbackMap callbacks_;
};
```

抽象工厂的好处在于可以根据模板派生出不同的具体工厂，并且任意产品只要在抽象工厂中进行注册，就可以生产该产品

### 建造者模式

当一个对象的构建比较复杂，我们需要将对象构造和表示进行分离

![](image-20220717154929298.png)

我们在建立一个复杂对象后，由于该对象属性众多，建立对象示例很不方便，于是我们将构造过程也写成一个类；为了生成不同的具有构造功能的类，我们还写一个指挥者类，用户**通过指挥者调用不同的建造者构造具体产品**。



例如我们要建造一座房子

```c++
class House
{
public:
    void setDoor(string door)
    {
        m_door = door;
    }
    void setWall(string wall)
    {
        m_wall = wall;
    }
    void setWindow(string window)
    {
        m_window = window;
    }

    string getDoor() { return m_door; }
    string getWall() { return m_wall; }
    string getWindow() { return m_window; }

private:
    string m_door;
    string m_wall;
    string m_window;
};
```

虽然可以直接通过 set 方法设定房子的结构

```c++
House *house = new House;
house->setDoor("门");
house->setWall("墙");
house->setWindow("窗");
```

但更好的是通过建造者构建，从而隐藏建造过程

```c++
class Builder
{
public:
    Builder()
    {
        m_house = new House;
    }
    
    void build(House *house)
    {
        buildDoor(house);
        buildWall(house);
        buildWindow(house);
    }

    void buildDoor(House *house)
    {
        house->setDoor("门");
    }
    void buildWall(House *house)
    {
        house->setWall("墙");
    }
    void buildWindow(House *house)
    {
        house->setWindow("窗");
    }

    House *getHouse() { return m_house; }

private:
    House *m_house;
};
```

这样我们通过如下方式建造房子：

```c++
Builder *builder = new Builder;
builder->build();
House *house = builder->getHouse();
```



但是上述方式中 builder 仅仅能建造一种房子，要实现泛化，就需要引入抽象建造类

```c++
// 建造者抽象类
class Builder
{
public:
    virtual void buildWall() = 0;
    virtual void buildWindow() = 0;
    virtual void buildDoor() = 0;
    House* getHouse() { return m_house; }

protected:
    House *m_house;
};

// 公寓工程队
class FlatBuilder : public Builder
{
public:
    FlatBuilder()
    {
        m_house = new House;
    }

    virtual void buildDoor()
    {
        m_house->setDoor("flat door");
    }
    virtual void buildWall()
    {
        m_house->setWall("flat wall");
    }
    virtual void buildWindow()
    {
        m_house->setWindow("falt window");
    }
};

// 别墅工程队
class VillaBuilder : public Builder
{
public:
    VillaBuilder()
    {
        m_house = new House;
    }

    virtual void buildDoor()
    {
        m_house->setDoor("villa door");
    }
    virtual void buildWall()
    {
        m_house->setWall("villa wall");
    }
    virtual void buildWindow()
    {
        m_house->setWindow("villa window");
    }
};
```

然后构造设计师类来管理不同的建造者

```c++
class Director
{
public:
    Director(Builder *builder) : m_builder(builder) {}

    void Construct()
    {
        m_builder->buildDoor();
        m_builder->buildWall();
        m_builder->buildWindow();
    }

private:
    Builder *m_builder;
};
```

我们就可以通过如下流程建造房屋：

```c++
// 请工程队
Builder *villaBuilder = new VillaBuilder;
// 设计师指挥建造
Director *director = new Director(villaBuilder);
director->Construct();
// 获得房屋
House *house = villaBuilder->getHouse();
```



### 原型模式

原型模式主要面对的问题是某些结构复杂的对象的创建工作；由于需求的变化，这些对象经常面临剧烈的变化，但是却拥有比较稳定一致的接口。**原型模式适用于复杂对象，其一个具有自我复制功能，且有统一的接口**。

![](image-20220717163953479.png)

```c++
class Person
{
public:
    virtual Person *clone() = 0;
    virtual void printT() = 0;
};

class CPPProgrammer : public Person
{
public:
    CPPProgrammer() {}
    
    CPPProgrammer(string name, int age)
    {
        m_name = name;
        m_age = age;
    }

    virtual void printT()
    {
        cout << "m_name: " << m_name << "m_age: " << m_age << endl;
    }
    
    virtual Person *clone()
    {
        CPPProgrammer *tmp = new CPPProgrammer;
        *tmp = *this;
        return tmp;
    }

private:
    string m_name;
    int m_age;
};
```



## 结构型模式

### 代理模式

代理 Proxy 是指具有与代理员（被代理的对象）具有相同接口的类，客户端必须通过代理与被代理的目标类交互，而代理一般在交互的过程中进行某些特别的处理。

![](image-20220717201214254.png)

```c++
class Subject
{
public:
    virtual void sailBook() = 0;
};

class RealSubjectBook : public Subject
{
public:
    virtual void sailBook()
    {
        cout << "SailBook" << endl;
    }
};

// 代理包含书店，同时实现协议 discount()
class DangDangProxy : public Subject
{
public:
    virtual void sailBook()
    {
        RealSubjectBook *rsb = new RealSubjectBook;
        discount();
        rsb->sailBook();
        discount();
    }

    void discount()
    {
        cout << "Discount" << endl;
    }

private:
    Subject *m_subject;
};
```



### 装饰模式

装饰 Decorator 模式又叫做包装模式，通过一种对客户端透明的方式来扩展对象的功能，是继承关系的一个替换方案。装饰模式就是要**把添加的附加功能分别放在单独的类中，并让这个类包含它要装饰的对象**，当需要执行时，客户端就可以有选择地、按顺序地使用装饰功能包装对象。

![](image-20220717202642827.png)

**装饰模式动态地给一个对象添加一些额外的职责，就增加功能而言，此模式比生成子类更加灵活。**

```c++
class Car
{
public:
    virtual void show() = 0;
};

class RunCar : public Car
{
public:
    virtual void show()
    {
        cout << "RunCar" << endl;
    }
};

class SwimCarDirector : public Car
{
public:
    SwimCarDirector(Car *car)
    {
        m_car = car;
    }

    void swim()
    {
        cout << "SwimCar" << endl;
    }

    virtual void show()
    {
        m_car->show();
        swim();
    }

private:
    Car *m_car;
};

class FlyCarDirector : public Car
{
public:
    FlyCarDirector(Car *car)
    {
        m_car = car;
    }

    void fly()
    {
        cout << "FlyCar" << endl;
    }

    virtual void show()
    {
        m_car->show();
        fly();
    }

private:
    Car *m_car;
};
```

我们用 Director 类装饰 Car 类，可以得到如下效果：

```c++
Car *car = new RunCar;
SwimCarDirector *swimcar = new SwimCarDirector(car);
FlyCarDirector *flycar = new FlyCarDirector(swimcar);

car->show();		// 会跑的车
swimcar->show();	// 会跑会游的车
flycar->show();		// 会跑会游会飞的车
```



### 适配器模式

适配器模式 Adapter 可以改变已有类或外部类的接口形式，使得原本由于接口不兼容而不能一起工作的类可以一起工作。

![](image-20220717204405541.png)

```c++
// 标准 18v 电
class Current18v
{
public:
    virtual void useCurrent18v() = 0;
};

// 使用 220v 电
class Current220v
{
public:
    void useCurrent220v()
    {
        cout << "Use 220v" << endl;
    }
};

// 适配器使得 18v 可以适配 220v
class Adapter : public Current18v
{
public:
    Adapter(Current220v *current)
    {
        m_current = current;
    }
    
    virtual void useCurrent18v()
    {
        cout << "Adapter 220v" << endl;
        m_current->useCurrent220v();
    }

private:
    Current220v *m_current;
};
```

于是我们可以使用如下代码：

```c++
Current220v *current220v = new Current220v;
Adapter *adapter = new Adapter(current220v);	// 适配 220 v
adapter->useCurrent18v();						// 调用 18v 电，实际上使用 220v
```



### 桥接模式

桥接 Bridge 模式基于类的最小设计原则，通过使用封装、聚合以及继承等行为来让不同的类承担不同的责任。它的主要特点是把抽象与行为实现分离开来，从而可以保持各部分的独立性以及应对它们的功能扩展。

![](image-20220717210336588.png)



例如我们可以定义发动机类和汽车类，然后汽车类保留发动机的指针，从而实现汽车可以安装不同的发动机

```c++
class Engine
{
public:
    virtual void InstallEngine() = 0;
};

class Engine4400cc : public Engine
{
public:
    virtual void InstallEngine()
    {
        cout << "Engine4400cc Installed" << endl;
    }
};

class Engine4500cc : public Engine
{
public:
```

### 组合模式

组合 Composite 模式，使用递归手段构造树形的对象结构，并可以用一个对象来访问整个对象树

![](image-20220717234246513.png)

例如我们设计一个文件和目录树，在不同目录下可以存放不同的目录和文件

```c++
#include <iostream>
#include <string>
#include <list>

using namespace std;

// 抽象文件和目录节点
class IFile
{
public:
    virtual void display() = 0;
    virtual int add(IFile *iflie) = 0;
    virtual int remove(IFile *iflie) = 0;
    virtual list<IFile *> *getChild() = 0;
};

// 文件节点
class File : public IFile
{
public:
    File(string name)
    {
        m_name = name;
    }

    virtual void display()
    {
        cout << m_name << endl;
    }
    virtual int add(IFile *iflie)
    {
        return -1;
    }
    virtual int remove(IFile *iflie)
    {
        return -1;
    }
    virtual list<IFile *> *getChild()
    {
        return NULL;
    }

private:
    string m_name;
};

// 目录节点
class Dir : public IFile
{
public:
    Dir(string name)
    {
        m_name = name;
        m_list = new list<IFile *>;
        m_list->clear();
    }
    ~Dir()
    {
        if (m_list != NULL)
        {
            for (auto it = m_list->begin(); it != m_list->end(); it++)
            {
                delete *it;
            }
        }
        delete m_list;
    }

    virtual void display()
    {
        cout << m_name << endl;
    }
    virtual int add(IFile *iflie)
    {
        m_list->push_back(iflie);
        return 0;
    }
    virtual int remove(IFile *iflie)
    {
        m_list->remove(iflie);
        return 0;
    }
    virtual list<IFile *> *getChild()
    {
        return m_list;
    }

private:
    string m_name;
    list<IFile *> *m_list;
};

// 递归显示树
void showTree(IFile *root, int level = 0)
{
    if (root == NULL)
    {
        return;
    }

    // 显示根节点
    root->display();
    list<IFile *> *rlist = root->getChild();

    // 不为 NULL 说明是目录，遍历目录中的文件
    if (rlist != NULL)
    {
        for (auto it = rlist->begin(); it != rlist->end(); it++)
        {
            // 输出空格
            for (int i = 0; i < level + 1; i++)
            {
                cout << "  ";
            }

            // 为 NULL 说明是文件，直接输出
            if ((*it)->getChild() == NULL)
            {
                (*it)->display();
            }
            // 否则是目录，递归输出
            else
            {
                showTree(*it, level + 1);
            }
        }
    }
}

int main(int argc, char **argv)
{
    Dir *root = new Dir("Root");
    Dir *dir1 = new Dir("Dir1");
    Dir *dir2 = new Dir("Dir2");
    File *file1 = new File("File1");
    File *file2 = new File("File2");
    File *file3 = new File("File3");
    File *file4 = new File("File4");

    dir1->add(file1);
    dir1->add(file2);

    dir2->add(file3);
    dir2->add(file4);

    root->add(dir1);
    root->add(dir2);

    showTree(root);

    return 0;
}
```

```shell
    File1  
    File2  
  Dir2     
    File3  
    File4
```

### 享元模式

享元 Flyweight 模式通过与其它类似对象共享数据来减小内存占用。

![](image-20220718005008528.png)

通过享元工厂产生享元对象，由享元对象创建不同的具体对象。

```c++
class Person
{
public:
    Person(string name, int age)
    {
        m_name = name;
        m_age = age;
    }
    virtual void printT() = 0;

protected:
    string m_name;
    int m_age;
};

class Teacher : public Person
{
public:
    Teacher(string name, int age, string id) : Person(name, age), m_id(id) {}

    void printT()
    {
        cout << "name: " << m_name << " age: " << m_age << " id: " << m_id << endl;
    }

private:
    string m_id;
};

// 享元工厂
class FlyweightFactory
{
public:
    FlyweightFactory()
    {
        m_map.clear();
    }
    ~FlyweightFactory()
    {
        // cycle and always delete the first element
        while (!m_map.empty())
        {
            Person *tmp = nullptr;
            auto it = m_map.begin();
            tmp = it->second;
            m_map.erase(it);    // remove the first element
            delete tmp;
        }
    }

    Person *GetTeacher(string id)
    {
        auto it = m_map.find(id);
        if (it == m_map.end())
        {
            string name;
            int age;
            cout << "Input Teacher's name:" << endl;
            cin >> name;
            cout << "Input Teacher's age:" << endl;
            cin >> age;

            Person *teacher = new Teacher(name, age, id);
            m_map.insert(make_pair(name, teacher));
            return teacher;
        }
        return it->second;
    }

private:
    map<string, Person *> m_map;
};
```

注意它和对象工厂的重要区别在于它会存储产生的对象，通过 map 的方式创建不存在的实例和返回存在的实例。

### 模板模式

```c++
class MakeCar
```

### 命令模式

命令模式通过 Command 类封装对目标对象的调用行为以及调用参数。在面向对象的程序设计中，一个对象调用另一个对象，一般情况下的调用过程是：创建目标对象实例、设置调用参数、调用目标对象的方法。但在**有些情况下有必要使用一个专门的类对这种调用过程加以封装，我们把这种专门的类称作 command 类**。

![](image-20220718103152120.png)

这种模式适用于将一个请求封装为一个对象，从而可以用不同的请求对客户端进行参数化；对请求排队或记录请求日志，以及支持可撤销的操作。

```c++
class Doctor
{
public:
    void treat_eye()
    {
        cout << "Doctor::treat_eye" << endl;
    }
    void treat_nose()
    {
        cout << "Doctor::treat_nose" << endl;
    }
};

// 指令提交给医生，医生治病
class Command
{
public:
    virtual void treat() = 0;

protected:
    Doctor *m_doctor;
};

class CommandTreatEye : public Command
{
public:
    CommandTreatEye(Doctor *doctor)
    {
        m_doctor = doctor;
    }
    void treat()
    {
        m_doctor->treat_eye();
    }
};

class CommandTreatNose : public Command
{
public:
    CommandTreatNose(Doctor *doctor)
    {
        m_doctor = doctor;
    }
    void treat()
    {
        m_doctor->treat_nose();
    }
};

// 护士持有指令
class BeautyNurse
{
public:
    BeautyNurse(Command *command)
    {
        m_command = command;
    }

    // 提交指令，医生看病
    void SubmittedCase()
    {
        m_command->treat();
    }

private:
    Command *m_command;
};

// 护士长，可以批量下达命令
class HeadNurse
```

### 责任链模式

此模式构造一系列分别担当不同职责的类的对象来共同完成一个任务，这些类的对象之间像链条一样，紧密相连。

![](image-20220718105355442.png)

例如，我们要建造汽车的不同部分

```c++
class CarHandle
{
public:
    virtual void HandleCar() = 0;
};

class HeadCarHandle : public CarHandle
{
public:
    virtual void HandleCar()
    {
        cout << "Make Head" << endl;
    }
};

class BodyCarHandle : public CarHandle
{
public:
    virtual void HandleCar()
    {
        cout << "Make Body" << endl;
    }
};

class TailCarHandle : public CarHandle
{
public:
    virtual void HandleCar()
    {
        cout << "Make Tail" << endl;
    }
};
```

使用这种代码虽然可以实现目标，但是建造过程会出现在客户端，如果我们要调整建造过程就必须修改客户端。



我们将命令按照链条的形式连接起来，从而实现任务传递

```c++
// 造完车后将任务传递下去
class CarHandle
{
public:
    CarHandle() : m_handle(NULL) {}

    virtual void HandleCar() = 0;
    CarHandle *setNextHandle(CarHandle *handle)
    {
        m_handle = handle;
        return m_handle;
    }

protected:
    CarHandle *m_handle; // 下一个处理单元
};

class HeadCarHandle : public CarHandle
{
public:
    virtual void HandleCar()
    {
        cout << "Make Head" << endl;
        if (m_handle != NULL)
        {
            m_handle->HandleCar();
        }
    }
};

class BodyCarHandle : public CarHandle
{
public:
    virtual void HandleCar()
    {
        cout << "Make Body" << endl;
        if (m_handle != NULL)
        {
            m_handle->HandleCar();
        }
    }
};

class TailCarHandle : public CarHandle
{
public:
    virtual void HandleCar()
    {
        cout << "Make Tail" << endl;
        if (m_handle != NULL)
        {
            m_handle->HandleCar();
        }
    }
};
```

于是我们得到如下业务逻辑

```c++
CarHandle *carhandle = new CarHandle;
CarHandle *headcarhandle = new HeadCarHandle;
CarHandle *bodycarhandle = new HeadCarHandle;
CarHandle *tailcarhandle = new HeadCarHandle;

// 设置建造顺序
carhandle->setNextHandle(headcarhandle);
carhandle->setNextHandle(bodycarhandle);
carhandle->setNextHandle(tailcarhandle);

// 开始建造
carhandle->HandleCar();
```

这样我们只需要调用一次建造函数，就可以按指定顺序执行任务。



### 策略模式

策略模式将算法的逻辑抽象接口 DoAction 封装到一个类 Context 中，再通过委托的方式将具体的算法实现委托给 Strategy 类。

![](image-20220718130058279.png)

它主要用于处理平滑的算法切换。



在编写具有不同功能的继承类时，可能需要提前规划需求来进行设计，我们先举一个例子：

```cpp
class Duck
{
public:
	Duck() {}
	virtual void fly() = 0;
	virtual void quack() = 0;
	virtual void swim() = 0;

protected:
	int status;
};

class BlackDuck : public Duck
{
public:
	BlackDuck() {}
	void fly() {}
	void quack() {}
	void swim() {}
};

class ToyDuck : public Duck
{
public:
	ToyDuck() {}
	void fly() {}
	void quack() {}
	void swim() {}
};
```

在上面的代码中，我们声明了一个 Duck 超类，其中包含 fly 等动作函数的接口，然后让不同的 Duck 子类继承，根据需求分别实现不同的动作，最终得到不同类型的鸭子对象。但是注意到玩具鸭 ToyDuck 不应该会飞，但因为 Duck 中声明了 fly ，我们不得不实现一个空的 fly 函数，这就造成了麻烦。更糟的是，如果还要给 Duck 添加更多的功能，就必须修改所有的子类，这一过程中又可能出现不需要的空函数，降低了代码维护的效率。



我们可以换一种思路：将鸭子的行为从 Duck 中分离出来单独作为一个类，这样 Duck 类不需要知道它们行为的实现细节。关键点是运用对 supertype 的多态性编程实现运行时对象可以动态变化，而不是被编码所固定。例如：

```cpp
class FlyBehavior
{
public:
	virtual void fly() = 0;
};

class FlyWithWings : public FlyBehavior
{
public:
	void fly()
	{
		cout << "Flying." << endl;
	}
};

class FlyNoWay : public FlyBehavior
{
public:
	void fly()
	{
		cout << "Cannot fly." << endl;
	}
};
```

我们定义一个抽象类作为飞行行为的接口，然后继承实现不同的飞行方法。



通过这种方式，我们修改 Duck 类为

```cpp
class Duck
{
public:
	Duck() {}
	void fly()
	{
		flybe->fly();
	}
	void setFly(FlyBehavior *f) { flybe = f; }

private:
	int status;
	FlyBehavior *flybe;
};
```

这样一来，子类中不需要再定义相关行为，直接通过基类函数调用，并且还可以方便地通过 setFly 调整飞行行为。

### 观察者模式

观察者模式定义对象之间一种一对多的依赖关系，当每一个对象改变状态，则所有依赖它们的对象都会得到通知。

![](image-20220718154653901.png)

```c++
class Secretary;

// 观察者
class PlayerObserver
{
public:
    PlayerObserver(Secretary *secretary)
    {
        m_secretary = secretary;
    }

    void update(string action)
    {
        cout << "action:" << action << endl;
    }

private:
    Secretary *m_secretary;
};

// 秘书监控外部变化，将结果通知给观察者
class Secretary
{
public:
    Secretary()
    {
        m_list.clear();
    }

    void Notify(string info)
    {
        // 给所有观察者发送信息
        for (auto it = m_list.begin(); it != m_list.end(); it++)
```

这样只要给每个观察者配置一个秘书类，就可以通过秘书类调用 notify 函数通知所有观察者。



### 备忘录模式

如果一个对象需要保持状态并可通过 undo 等操作恢复到以前的状态时，就可以使用 Memento 模式。

```c++
class Memento
{
public:
    Memento(string name, int age)
    {
        m_age = age;
        m_name = name;
    }

    string getName() const { return m_name; }
    int getAge() const { return m_age; }

    void setName(string name) { m_name = name; }
    void setAge(int age) { m_age = age; }

private:
    string m_name;
    int m_age;
};

class Person
{
public:
    Person(string name, int age)
    {
        m_age = age;
        m_name = name;
    }

    string getName() const { return m_name; }
    int getAge() const { return m_age; }

    void setName(string name) { m_name = name; }
    void setAge(int age) { m_age = age; }

    // 创建备忘录
    Memento *createMemento()
    {
        return new Memento(m_name, m_age);
    }
    // 恢复
    void setMemento(Memento *memento)
    {
        m_age = memento->getAge();
        m_name = memento->getName();
    }

private:
    string m_name;
    int m_age;
};

// 管理备忘录
class Caretaker
{
public:
    Caretaker(Memento *memento)
    {
        m_memento = memento;
    }

    Memento *getMemento() { return m_memento; }
    void setMemento(Memento *memento) { m_memento = memento; }

    Memento *m_memento;
```

### 访问者模式

访问者模式分离对象的数据和行为，可以在不修改已有类的情况下，增加新的操作角色和职责。

![](image-20220718162555662.png)

```c++
class ParkElement;

class Visitor
{
public:
    virtual void visit(ParkElement *parkelement) = 0;
};

// 公园元素
class ParkElement
{
public:
    virtual void accept(Visitor *visitor) = 0;
};

class ParkA : public ParkElement
{
public:
    // 公园接受访问者的访问
    virtual void accept(Visitor *visitor)
    {
        visitor->visit(this);
    }
};

class ParkB : public ParkElement
{
public:
    // 公园接受访问者的访问
    virtual void accept(Visitor *visitor)
    {
        visitor->visit(this);
    }
};

// 整个公园
class Park : public ParkElement
{
public:
    Park()
    {
        m_list.clear();
    }
    void setParkElement(ParkElement *element)
    {
        m_list.push_back(element);
    }

    virtual void accept(Visitor *visitor)
    {
        // 公园的每一部分接受访问
        for (auto it = m_list.begin(); it != m_list.end(); ++it) {
            (*it)->accept(visitor);
        }
    }

private:
    list<ParkElement *> m_list;
};

// 清洁工
class VisitorA : public Visitor
{

    {
        cout << "VisitorManager clean the Park" << endl;
    }
};
```

### 状态模式

状态 State 模式允许通过改变对象的内部状态而改变对象的行为，这个对象表现得好像修改了它的类一样。状态模式主要解决的是当控制一个对象状态转换的条件表达式过于复杂时的情况。

![](image-20220718170508499.png)

```c++
class Worker;

class State
{
public:
    virtual void doThing(Worker *worker) = 0;
};

class Worker
{
public:
    int getHour()
    {
        return m_hour;
    }
    State *getState()
    {
        return m_state;
    }

    void setHour(int hour)
    {
        m_hour = hour;
    }
    void setState(State *state)
    {
        m_state = state;
    }

    // Worker 根据状态 State 决定做什么事
    void doThing()
    {
        m_state->doThing(this);
    }

private:
    int m_hour;
    State *m_state;
};

class State1 : public State
{
public:
    void doThing(Worker *worker)
    {
        if (worker->getHour() == 7 || worker->getHour() == 8)
        {
            cout << "Breakfast" << endl;
        }
        // 不满足状态就改变状态
        else
        {
            delete worker->getState();
            worker->setState(new State2);
            worker->getState()->doThing(worker);
        }
    }
};

class State2 : public State
{
public:
    void doThing(Worker *worker)
    {
        if (worker->getHour() == 9 || worker->getHour() == 10)
        {
            cout << "Work" << endl;
        }
        // 不满足状态就改变状态
        else
        {
            delete worker->getState();
            worker->setState(new State1);
            worker->getState()->doThing(worker);
        }
    }
};
```



### 解释模式

解释模式提供一个实现语法解释器的框架，通过这个解释器来解释语言中的句子。

![](image-20220718172018676.png)

```c++
class Context
{
public:
    Context(int num, int res) : num(num), res(res) {}

    int getNum() const { return num; }
    int getRes() const { return res; }

    void setNum(int num) { this->num = num; }
    void setRes(int res) { this->res = res; }

private:
    int num;
    int res;
};

class Expression
{
public:
    virtual void interpret(Context *context) = 0;
};

class PlusExpression : public Expression
{
public:
     virtual void interpret(Context *context)
     {
        int num = context->getNum();
        num++;
        context->setRes(num);
     }
};

class MinusExpression : public Expression
{
public:
     virtual void interpret(Context *context)
     {
        int num = context->getNum();
        num--;
        context->setRes(num);
     }
};
```

实际上这种模式似乎类似于策略模式，我们通过不同的表达式类解析内容，得到不同的结果。



### 迭代器模式

迭代器模式就是在 STL 标准库中广泛应用的访问容器的重要方式，通过产生一个迭代器就可以直接获取容器中的元素。

![](image-20220718213627291.png)

```c++
using Object = int;
#define SIZE 255

// 抽象迭代器类
class MyIterator
{
public:
    virtual void First() = 0;
    virtual void Next() = 0;
    virtual bool isDone() = 0;
    virtual Object CurrentItem() = 0;
};

// 抽象集合
class Aggregate
{
public:
    virtual MyIterator *CreateIterator() = 0;
    virtual Object getItem(int index) = 0;
    virtual int getSize() = 0;
};

class ConcreteIterator : public MyIterator
{
public:
    // 迭代器获得集合的指针
    ConcreteIterator(Aggregate *ag)
    {
        ag_ = ag;
        currentIndex_ = 0;
    }

    virtual void First()
    {
        currentIndex_ = 0;
    }
    virtual void Next()
    {
        if (currentIndex_ < ag_->getSize())
            currentIndex_++;
    }
    virtual bool isDone()
    {
        return currentIndex_ == ag_->getSize();
    }
    virtual Object CurrentItem()
    {
        return ag_->getItem(currentIndex_);
    }

private:
    int currentIndex_;
    Aggregate *ag_;
};

class ConcreteAggregate : public Aggregate
{
public:
    ConcreteAggregate()
    {
        for (int i = 0; i < SIZE; i++)
        {
            object[i] = i + 100;
        }
    }

    // 创建迭代器
    virtual MyIterator *CreateIterator()
    {
        return new ConcreteIterator(this);
    }
    virtual Object getItem(int index)
    {
        return object[index];
    }
    virtual int getSize()
    {
        return SIZE;
    }

private:
    Object object[SIZE];
};
```
