### 设计模式

#### 工厂模式

通过工厂模式，借助命令名称来创建指令；着色器工厂，目的之一是分离模块。`AIS_Object` 要保存着色器指针，并且要调用 `draw` 方法。要为物体设置着色器，就必须在创建它之后，为它设定着色器，这就可能需要引入具体着色器类的头文件，而且每次都要创建着色器很麻烦。为了分离模块，通过一个工厂在 `AIS_Object` 内部，根据自身类型传入类型名自动创建相应的着色器。这个创建方法是从外部传入的，它返回基类 `AIS_Renderer` 指针，因此不需要在 `AIS_Object` 头文件中引入头文件，只需要在 `GL` 模块中初始化时设定这个创建方法，这样每次 `AIS_Object` 实例化时自动调用这个方法。



#### 策略模式

用来为相同的算法目标应用不同的算法策略。



#### 观察者模式

借助 CRTP 模式结合观察者模式进行消息处理。

- `AIS_Object` 可以接收任何消息，例如一列 `AIS_MsgImpl` 的子类消息
- 然后 `AIS_Object->accept` 会调用 `AIS_MsgImpl` 的 `accept` 方法，传入自身以及所有子节点
- `visit` 方法内部知道消息自身的类型（通过模板类型），因此可以将自身强制转换后调用 `AIS_Object->visit` 进入对应消息的处理函数

```cpp
struct AIS_Msg
{
    /**
     * @brief 访问接口
     *
     * @param[in] visitor
     */
    virtual void accept(AIS_Object *visitor) = 0;

    /**
     * @brief 虚析构函数
     *
     */
    virtual ~AIS_Msg() = default;
};

template <class Derived> struct AIS_MsgImpl : AIS_Msg
{
    /**
     * @brief CRTP 模式
     *
     * @param[in] visitor
     */
    void accept(AIS_Object *visitor) override;
};

class EXT_SYMBOL AIS_Object
{
  public:
    /**
     * @brief 消息处理函数
     *
     * @param[in] msg
     */
    void handle_msg(std::shared_ptr<AIS_Msg> msg);

    /**
     * @brief 访问接口
     *
     * @param[in] msg
     */
    virtual void visit(AIS_InitMsg *msg);
    virtual void visit(AIS_UpdateMsg *msg);
    virtual void visit(AIS_PaintMsg *msg);
    virtual void visit(AIS_ShowMsg *msg);
    virtual void visit(AIS_MoveMsg *msg);
    virtual void visit(AIS_MouseMsg *msg);
    virtual void visit(AIS_DestroyMsg *msg);

  protected:
    std::vector<std::shared_ptr<AIS_Object>> m_children;
    std::shared_ptr<AIS_Renderer> m_renderer;
};

// 利用静态转换，让 visit 匹配不同的 AIS_Object 中的重载函数
template <class Derived> inline void AIS_MsgImpl<Derived>::accept(AIS_Object *visitor)
{
    // 静态判断，要求继承关系，节省运行时开销
    static_assert(std::is_base_of<AIS_MsgImpl, Derived>::value, "Derived must be a Msg");
    visitor->visit(static_cast<Derived *>(this));
}
```
