    生成位图

    BMP（全称 Bitmap）是 Windows 操作系统中的标准图像文件格式，可以分成两类：设备相关位图（DDB）和设备无关位图（DIB），使用非常广。它采用位映射存储格式，除了图像深度可选以外，不采用其他任何压缩，因此，BMP文件所占用的空间很大。BMP 文件的图像深度可选 lbit、4bit、8bit 及 24bit 。BMP 文件存储数据时，图像的扫描方式是按从左到右、从下到上的顺序。由于 BMP 文件格式是 Windows 环境中交换与图有关的数据的一种标准，因此在 Windows 环境中运行的图形图像软件都支持 BMP 图像格式。

    位图保存格式似乎有问题，导致 150x150 规格的图像不能正常显示。

    使用 C++ 操作生成位图资源，首先声明位图资源和操作位图的函数

void build_bmp(std::string _file,
               int _w,
               int _h,
               char *_image)
{
    int width = _w;
    int height = _h;
    std::ofstream outFile(_file.c_str(), std::ios::binary);
    BITMAPFILEHEADER bmpFileHeader;
    BITMAPINFOHEADER bmpInfoHeader;
    bmpInfoHeader.biSize = sizeof(BITMAPINFOHEADER);
    bmpInfoHeader.biWidth = width;
    bmpInfoHeader.biHeight = height;
    bmpInfoHeader.biPlanes = 1;
    bmpInfoHeader.biBitCount = 24;
    bmpInfoHeader.biCompression = 0;
    bmpInfoHeader.biSizeImage = width * height * 3;
    bmpInfoHeader.biXPelsPerMeter = 2834;
    bmpInfoHeader.biYPelsPerMeter = 2834;
    bmpInfoHeader.biClrUsed = 0;
    bmpInfoHeader.biClrImportant = 0;

    ((char*)(&bmpFileHeader.bfType))[0] = 'B';
    ((char*)(&bmpFileHeader.bfType))[1] = 'M';
    bmpFileHeader.bfReserved1 = 0;
    bmpFileHeader.bfReserved2 = 0;
    bmpFileHeader.bfOffBits = bmpInfoHeader.biSize + sizeof(BITMAPFILEHEADER);
    bmpFileHeader.bfSize = bmpFileHeader.bfOffBits + bmpInfoHeader.biSizeImage;

    outFile.write(reinterpret_cast<char*>(&bmpFileHeader), sizeof(BITMAPFILEHEADER));
    outFile.write(reinterpret_cast<char*>(&bmpInfoHeader), sizeof(BITMAPINFOHEADER));

    outFile.write(_image, bmpInfoHeader.biSizeImage);
    outFile.close();
};

    曼德布洛集

    曼德布洛集 $M$ 是一个几何图形，曾被称为“上帝的指纹”。这个点集出自非线性迭代公式 $$ z_{n+1} = z_n^2+c $$ 所有使得无限迭代后的结果能保持有限数值的复数 $z$ 的集合连通的 $c$，构成曼德布洛集，也称该迭代函数的 Julia 集。

    最简单的生成代码为

    int N = 255;
double R = 4.0;
for (int j = 0; j < height; j++)
{
    for (int i = 0; i < width; i++)
    {
        auto x = (double)i / width * 3.5 - 2.5;
        auto y = (double)j / height * 2.0 - 1.0;
        auto c = std::complex<double>{x, y};
        auto z = std::complex<double>{0.0, 0.0};
        TGAColor color;

        // |z| < 2 时，认为属于集合
        int count = 0;
        double pow = 1.0;
        while (count++ < N)
        {
            double z2 = std::pow(std::abs(z), 2);
            if (z2 > R)
            {
                color.bgra[0] = 255;
                color.bgra[1] = 255;
                color.bgra[2] = 255;
                break;
            }

            z = z * z + c;
            pow *= 2;
        }

        painter->set_pixel(i, j, color);
    }
}

    采用另一种上色方式：对于发散的点，根据迭代奇数或偶数次决定颜色，得到

    势能函数

    定义势能函数 $$ V(c) = \lim_{ n \to \infty } \frac{\log_{+}|P^n_{c}(0))|}{2^n} $$ 其中 $\log_{+}=\max(0,\log),P_{c}(z)=z^2+c$，而 $P_{c}^n(z)=(P_{c}\circ\cdots \circ P_{c})(z)$，则右端极限存在。如果给定 $R$，迭代 $n$ 次时 $|z|>R$，则极限误差为 $O\left( \frac{1}{R} \right)$ 。显然，如果 $c\in M$，则 $V(c)=0$ 。对于 $z\in\mathbb{C}$，则 $V=z$ 是等势面。

    绘制等势面

    我们可以绘制这些等势面。给定 $R$，则 $n$ 需要很大才能使得 $|z|=R$，而此时 $V(c)\approx \frac{\log_{+}(R)}{2^n}$，因此我们可以根据 $V$ 的值计算颜色。例如使用周期函数 $$ x = \log(V),\quad color = \frac{1+\cos(2\pi x)}{2} $$

    计算代码为

    auto V = log(z2) / pow;
color.bgra[0] = 255 * (1 + cos(2 * Numeric::pi * log(V))) / 2;
color.bgra[1] = 255 * (1 + cos(2 * Numeric::pi * log(V))) / 2;
color.bgra[2] = 255 * (1 + cos(2 * Numeric::pi * log(V))) / 2;

    颜色缩放

    由于我们使用对数，因此 $$ \log(V) = \log \log(P) - n\log 2 $$ 其中 $n$ 只会产生平移。可以利用 $K>0$ 来控制颜色的深度。令 $$ \begin{aligned} x &= \frac{\log(V)}{K},\quad (R,G,B) = \left(\frac{1-\cos(ax)}{2}, \frac{1-\cos(bx)}{2},\frac{(1-\cos(cx))}{2} \right)\times 255\ (a,b,c) &= \left(1, \frac{1}{3\sqrt{ 2 }}, \frac{1}{7\cdot 3^{1/8}} \right) \times \frac{1}{\log 2} \end{aligned} $$ 其中 $(a,b,c)$ 是任取的。于是通过修改 $K$ 就可以调整颜色缩放。注意到 $-n\log 2$ 这一项，因此可以将 $K=\log 2$ 作为一个起始值，这样等势面颜色根据迭代次数平移而周期变换。计算代码

    // z = z^2 + c
int N = 2000;
double R = 1000000.0;
for (int j = 0; j < height; j++)
{
    for (int i = 0; i < width; i++)
    {
        auto x = (double)i / width * 3.5 - 2.5;
        auto y = (double)j / height * 2.0 - 1.0;
        auto c = std::complex<double>{x, y};
        auto z = std::complex<double>{0.0, 0.0};
        TGAColor color;

        // |z| < 2 时，认为属于集合
        int count = 0;
        double pow = 1.0;
        while (count++ < N)
        {
            double z2 = std::pow(std::abs(z), 2);
            if (z2 > R)
            {
                auto V = log(z2) / pow;
                double K = log(2);
                double a = 1 / log(2), b = a / (3 * std::pow(2, 1.0 / 2)), c = a / (7 * std::pow(3, 1.0 / 8));
                color.bgra[0] = 255 * (1 - cos(2 * Numeric::pi * log(V) / K * a)) / 2;
                color.bgra[1] = 255 * (1 - cos(2 * Numeric::pi * log(V) / K * b)) / 2;
                color.bgra[2] = 255 * (1 - cos(2 * Numeric::pi * log(V) / K * c)) / 2;
                break;
            }

            z = z * z + c;
            pow *= 2;
        }

        painter->set_pixel(i, j, color);
    }
}

    加速方法

    内部检测

    [!axioms] 作用 operation 满足 - $1_{G}*s=s,\forall s\in S$ - 结合律

    [!definition] 给定作用群 $G$ 和 $s\in S$，则 $O_{s}={s'\in S: \exists g\in G,gs=s' }$ 称为 $s$ 的轨道。

    [!definition] 若存在 $k>0$，使得 $P^k(a)=a$，则 $a$ 称为 $P$ 的周期点。最小的使得 $P^k(a)=a$ 的 $k$ 称为 $a$ 的周期，此时 $a$ 的轨道是有限的，称为一个循环，其中任意点处 $(P^k)'$ 都相同，称为循环的乘子。如果乘子模小于 1，则一个循环被称为是吸引的。如果 $P_c$ 具有吸引循环，则称参数 $c$ 是双曲的。

    由于 $P_{c}(z)=z^2+c$，Fatou 证明了双曲参数对应的临界点被吸引到 $P_c$ 的循环，并且 $P_{c}$ 至多具有一个吸引循环。特别地，$c\in M$ 。

    这是非局部全纯动力学的显著特征：循环具有达到临界点的吸引力。

    在更一般的动力系统中，双曲具有其它的定义，而在我们的语境下则表示 $P_{c}$ 被扩张到它的 Julia 集。双曲参数是稳定的：所有邻近的参数也是双曲的，并且当 $c$ 在一个小邻域中变动时，吸引循环和 Julia 集连续变化。对应 $c$ 的双曲分量是包含 $c$ 的所有双曲参数集合的连通分量。这一分量被一条除一点以外都光滑的有界代数曲线控制。这条曲线由具有模 1 乘子的循环对应的参数组成。对于 $P_{c}$，它的分量要么是圆盘，要么是心形的内部。

    可以证明这些边界曲线被包含在 $M$ 的拓扑边界中：因此一个双曲分量也是 $M$ 内部的连通分量，反之则未知（至今仍然是一个开放问题）。

    检查导数

    随着 $z$ 不断迭代，我们观察导数 $P'(z)=2z$ 。由于复合函数求导满足 $$ (P^n)'(z_{0}) = P'(P^{n-1}(z_{0}))\cdot P'(P^{n-2}(z_{0}))\cdots P'(z_{0}) = \prod_{i=0}^{n-1}P'(z_{n}) $$ 所以只需要不断累乘 $2z_i$ 就可以得到每步的导数。如果 $c$ 是一个双曲参数，即 $P_{c}$ 具有吸引循环，这意味着它的轨道 $c,P(c),P^2(c),\cdots$ 趋于一个吸引循环，乘子 $(P^k)'$ 模小于 1，因此导数将会趋于零。反之则未必，不过这些反例是稀疏的。

    基本想法

    我们知道双曲分量总是 $M$ 内部的连通分量，因此在检查 $|z|$ 的同时，还检查是否收敛到吸引循环，从而加速计算。选择一个阈值 $\epsilon$，如果导数小于阈值，则认为这个像素在 $M$ 中。

    // z = z^2 + c
int N = 2000;
double R = 1000000.0;
for (int j = 0; j < height; j++)
{
    for (int i = 0; i < width; i++)
    {
        auto x = (double)i / width * 3.5 - 2.5;
        auto y = (double)j / height * 2.0 - 1.0;
        auto c = std::complex<double>{x, y};
        auto z = c;
        auto der = std::complex<double>{1.0, 0.0};
        TGAColor color;

        for (int k = 0; k < N; k++)
        {
            // 内部，收敛到吸引循环
            if (std::abs(der) < 1e-3)
                break;

            if (std::abs(z) > R)
            {
                color.bgra[0] = 255;
                color.bgra[1] = 255;
                color.bgra[2] = 255;
                break;
            }

            der = der * z * std::complex<double>{2.0, 0.0};
            z = z * z + c;
        }

        painter->set_pixel(i, j, color);
    }
}

    这种判断方法可能把不是内部的点标记为内部，并且 $c$ 的轨道可能要迭代很多次才能接近零。对于第二种情况，直观上这类 $c$ 应该会接近双曲分量的中心；对于第一种情况，存在非双曲参数 $c$ 使得 $\lim_{ \to \infty }\inf_{n}(P^n)'(c)=0$，不过它们非常少。任何在 $M$ 外部的点都存在某个 $f>0$ 使得 $\inf(P^n)'(c)\ge f(V(c))$，因此至少被这个方法判定不属于 $M$ 的点实际上也会接近 $M$，通过减小阈值可以缩小这个距离。

    边界检测

    Milnor 准则

    在迭代 $P: z \mapsto z^2+c$ 中，令 $z_{0}=0,\rho_{n}= \frac{dz_{n}}{dc}$ 。选择一个足够大的阈值 $R>0$，以及 $n$ 使得 $|z_{n}|>R$，则从 $z$ 到 $M$ 的距离估计值为 $$ d_{n} = \frac{2|z_{n}|\log|z_{n}|}{\rho_{n}} $$ 序列 $\rho_{n}$ 可以从 $\rho_{0}=0$ 开始迭代 $$ \begin{aligned} \rho_{n} &= \frac{dz_{n}}{dc} = \frac{dP_{c}^n(z_{0})}{dc}= \frac{d([P_{c}^{n-1}(z_{0})]^2+c)}{dc}\ &= 2[P_{c}^{n-1}(z_{0})] \frac{dP_{c}^{n-1}(z_{0})}{dc}+1\ &= 2z_{n-1} \rho_{n-1}+1 \end{aligned} $$ 当 $n$ 趋于无穷时，近似值 $d_{n}$ 并不会收敛到 $z$ 到 $M$ 距离的真实值。事实上，只有弱不等关系 $$ \mathrm{given }\ |z|<5.7,\quad\frac{1}{5} < \frac{d}{\lim_{ n \to \infty } d_{n}}<1 $$ 下界可以被替换为关于 $|z|$ 的值，不过不会小于 $\frac{1}{4}$ 。当 $R>1000,|z_{n}|>R$ 时 $d_{n}$ 非常接近 $\lim_{ n \to \infty }d_{n}$ 。

    当迭代像素被判定为外部点时，我们检测 $d_{n}$ 是否小于给定的阈值，如果满足条件就认为当前像素是边界点，填充为边界颜色。

    // z = z^2 + c
int N = 2000;
double R = 1000000.0;
double thickness = 0.01;
for (int j = 0; j < height; j++)
{
    for (int i = 0; i < width; i++)
    {
        auto x = (double)i / width * 3.5 - 2.5;
        auto y = (double)j / height * 2.0 - 1.0;
        auto c = std::complex<double>{x, y};
        auto z = c;
        auto dPz = std::complex<double>{1.0, 0.0};
        auto dPc = std::complex<double>{1.0, 0.0};
        TGAColor color;

        bool outside = false;
        for (int k = 0; k < N; k++)
        {
            // 内部，收敛到吸引循环
            if (std::abs(dPz) < 1e-3)
                break;

            if (std::abs(z) > R)
            {
                color.bgra[0] = 255;
                color.bgra[1] = 255;
                color.bgra[2] = 255;
                outside = true;
                break;
            }

            dPz = dPz * z * std::complex<double>{2.0, 0.0};
            dPc = dPc * z * std::complex<double>{2.0, 0.0} + std::complex<double>{1.0, 0.0};
            z = z * z + c;
        }

        if (outside)
        {
            auto rsq = std::pow(std::abs(z), 2);
            if (rsq * std::pow(log(rsq), 2) < std::pow(thickness * std::abs(dPc), 2))
            {
                color.bgra[0] = 255;
                color.bgra[1] = 0;
                color.bgra[2] = 255;
            }
        }

        painter->set_pixel(i, j, color);
    }
}
