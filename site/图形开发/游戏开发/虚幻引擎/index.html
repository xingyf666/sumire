    虚幻引擎

    虚幻引擎基础

    安装配置

    VS 配置

    在 VS installer 中选择使用 C++ 的游戏开发安装

    还需要安装 .Net 6.0 组件

    Rider 配置

    首先需要在设置中安装 .Net 环境

    安装 RiderLink 用于链接 UE 和编辑器

    引擎配置

    安装号引擎后，可以找到“选项”修改安装组件

    可以安装调试工具（50 GB），这样可以直接在编辑器中打断点调试。

    启动引擎

    编译源码

    要访问 UE 官方的仓库，需要注册 Epic 账号并在 GitHub 上关联账号

    该错误出现在 BlackmagicCore 插件中，似乎在 Common.h 中遗漏了 chrono 头文件，添加头文件后即可正常编译。

    创建项目

    打开引擎，选择“游戏”，右侧选择“C++”，然后创建项目

    注意右下角的项目名称要改成英文!!！

    迁移项目

    打开引擎后，可能无法正常打开其它版本的项目。这时候创建一个空的项目（对于 Rider 需要修改"Edit-Editor Preferences"中的"Source Code Editor"为"Rider"，然后尝试打开 .uproject 文件

    此时即使无法正常打开，也会生成一个 .sln 项目文件，用 Rider 打开后，编译报错

    ChaosTestEditor modifies the values of properties: [ UndefinedIdentifierWarningLevel: Off != Error ]. This is not allowed, as ChaosTestEditor has build products in common with UnrealEditor. Remove the modified setting, change ChaosTestEditor to use a unique build environment by setting 'BuildEnvironment = TargetBuildEnvironment.Unique;' in the ChaosTestEditorTarget constructor, or set bOverrideBuildEnvironment = true to force this setting on. 

Result: Failed (OtherCompilationError)

    此时找到 ChaosTestEditor.Target.cs 文件，添加 BuildEnvironment 配置，重新编译即可

    public class ChaosTestEditorTarget : TargetRules  
{  
    public ChaosTestEditorTarget(TargetInfo Target) : base(Target)  
    {       Type = TargetType.Editor;  
       DefaultBuildSettings = BuildSettingsVersion.V5;  
       IncludeOrderVersion = EngineIncludeOrderVersion.Unreal5_5;  
       ExtraModuleNames.Add("ChaosTest");  

       // 添加这一行  
       BuildEnvironment = TargetBuildEnvironment.Unique;  
    }
}

    默认 C++ 类

    • 旁观者类 Spectator：第三方类

    选择“公共”，然后创建类

    类似地，分别创建

    • Pawn
    • HUD
    • PlayerController
    • PlayState
    • GameState

    这样就得到对应的头文件和源文件。

    宏的作用

    以生成的 GameMode 类为例

    UCLASS()
class PROJECT_API AMyGameMode : public AGameMode
{
    GENERATED_BODY()

};

    其中使用了 GENERATED_BODY 宏，表示不直接使用父类构造函数。不同宏的作用总结如下

    宏 作用
    GENERATED_BODY() 不直接使用父类构造函数，自定义初始化需要自己声明，并在 .cpp 中实现。注意它之后的成员都是私有成员
    GENERATED_UCLASS_BODY() 使用父类构造函数，自定义初始化可以直接在 .cpp 中实现，宏自动生成带有特定参数的构造函数。注意它之后的成员都是公有成员
    UCLASS() 生成类的反射数据，类必须派生自 UObject
    UPROPERTY 属性声明宏，用于反射系统
    UFUNCTION 函数声明宏，用于反射系统
    USTRUCT 结构体声明宏，用于反射系统
    UENUM 枚举声明宏，用于反射系统

    其中许多宏都用于反射系统，虚幻使用反射负责垃圾回收、引用更新、编辑器集成等高级功能。

    虚幻 C++ 基础

    基础变量

    常用的基本类型包括

    类型 含义 类型 含义
    FString 字符串 FName 名称
    FText 文本 FVector 三维向量
    FRotator 欧拉角 FTransform 组合平移、旋转、缩放

    其中

    • FString 专用于字符串操作，可以修改，性能较低；
    • FName 保存名称，不可修改，它在创建时自动生成哈希值，相同的内容只生成一次，因此作为键进行查找时效率很高；
    • FText 显示玩家可以直接看到的信息，用于多种语言的处理，不可修改，比较臃肿，但是利于本地化。

    三种类型的相互转换

    FString string = TEXT("String");

// string 转 name, text
FName name = FName(string);
FText text = FText::FromString(string);

// name, text 转 string
string = name.ToString();
string = text.ToString();

// name 转 text
text = FText::FromName(name);
name = FName(text.ToString());

    注意 FText 不能直接转换为 FName，需要先转换为 FString 。

    TArray

    TArray 是虚幻中的动态数组，速度快，内存消耗小，安全性高，其中的元素类型必须相同。使用示例

    TArray<int32> MyArray;
MyArray.Add(10);
MyArray.Add(20);
MyArray.Add(30);
MyArray.Add(40);
MyArray.AddUnique(10);

for (auto val : MyArray)
{
    UE_LOG(LogTemp, Warning, TEXT("%d"), val);
    GEngine->AddOnScreenDebugMessage(-1, 0.5f, FColor::Blue, FString::Printf(TEXT("%d"), val));
}

    TMap

    TMap 是虚幻中的键值对容器。使用示例

    TMap<int32, int32> MyMap;
MyMap.Emplace(10, 12);
MyMap.Emplace(11, 13);
MyMap.Emplace(12, 14);
MyMap.Emplace(13, 15);

for (auto&& [key, val] : MyMap)
{
    UE_LOG(LogTemp, Warning, TEXT("%d->%d"), key, val);
    GEngine->AddOnScreenDebugMessage(-1, 0.5f, FColor::Blue, FString::Printf(TEXT("%d->%d"), key, val));
}

    可以分别获得键和值的列表

    TArray<int32> Keys, Vals;
MyMap.GenerateKeyArray(Keys);
MyMap.GenerateValueArray(Vals);

    另外还有 TMultiMap 类型，允许多个重复键。

    TSet

    TSet 是虚幻中的快速容器类，用于顺序不重要的元素存储。使用示例

    TSet<FString> MySet;
MySet.Add(TEXT("Banana"));
MySet.Add(TEXT("Apple"));

// Emplace 原地构造
MySet.Emplace("Pair");
MySet.Emplace("Watermelon");

for (auto&& val : MySet)
{
    UE_LOG(LogTemp, Warning, TEXT("%s"), *val);
    GEngine->AddOnScreenDebugMessage(-1, 0.5f, FColor::Blue, FString::Printf(TEXT("%s"), *val));
}

    注意需要通过 * 获得 FString 中的字符串，才能与 %s 匹配。

    可以将 TSet 中的元素存到一个数组中

    TArray<FString> array = MySet.Array();

    还可以排序

    MySet.Sort([](auto& A, auto& B) {
    return A.Len() > B.Len();
});

    通过 [] 获得元素的引用，可以修改对应的元素

    // 在添加元素时获得索引
FSetElementId index = MySet.Add(TEXT("Purple"));
MySet[index] = TEXT("Orange");

    可以从整数构建索引

    for (int i = 0; i < MySet.Num(); i += 2)
    MySet.Remove(FSetElementId::FromInteger(i));

// 删除末端的空闲元素

    还可以将所有空闲元素移动到末尾，然后删除

    • UPROPERTY(EditInstanceOnly) 只能在实例化细节面板编辑

    • UPROPERTY(Transient) 不会被序列化保存，用于临时数据或不希望保存的数据

    UPROPERTY(EditAnywhere, BlueprintReadOnly)

    变量分组

    可以对变量分组，使用

    UPROPERTY(VisibleAnywhere, Category = "MyIntValue")

    这样在面板中显示时，就会按照目录分组。

    元数据说明符

    通过 meta 指定元数据。例如变量显示名

    UPROPERTY(VisibleAnywhere, meta = (DisplayName = "MyValueDisplayName"))

    可以指定变量的编辑条件

    UPROPERTY(VisibleAnywhere, meta = (DisplayName = "Controller"))
bool isController;

UPROPERTY(VisibleAnywhere, meta = (EditCondition = "isController"))

    此时只有当 isController 为 true 时才可以编辑 value 值。

    可以给出变量的解释

    UPROPERTY(VisibleAnywhere, meta = (Tooltip = "This value is true/false."))
bool value;

    此时将鼠标悬停在变量上，可以看到变量的解释

    可以让变量显示在蓝图节点上

    UPROPERTY(BlueprintReadWrite, meta = (ExposeOnSpawn = "MyValue"))
bool MyValue;

    以及允许蓝图访问私有成员

    UPROPERTY(BlueprintReadWrite, meta = (AllowPrivateAccess = "true"))
bool MyValue;

    UFUNCTION 宏

    定义特殊函数

    首先是指定在 C++ 中实现，可在蓝图中调用的函数

    // 在 C++ 中实现，在蓝图中调用
UFUNCTION(BlueprintCallable, Category = "MyFunction")
void PrintF1();

// 在 C++ 中实现，在蓝图中调用
UFUNCTION(BlueprintCallable, BlueprintPure, Category = "MyFunction")
bool PrintF2();

    其中 BlueprintPure 表示纯函数，它可以在蓝图中直接调用，不需要与其它节点连接。

    还有只能声明，需要在蓝图中重写的函数

    // 在 C++ 中声明，不可定义，在蓝图中重写
UFUNCTION(BlueprintImplementableEvent)
void Test1();

UFUNCTION(BlueprintImplementableEvent)
int Test2();

UFUNCTION(BlueprintImplementableEvent)
void Test3(int);

UFUNCTION(BlueprintImplementableEvent)
int Test4(int);

    还可以指定可以 C++ 实现，可以在蓝图中重写或不重写的函数

    UFUNCTION(BlueprintNativeEvent)
void Test();

    其中有返回值的函数在蓝图中是函数，而无返回值的函数在蓝图中是事件。

    if (Controller)

        const FRotator YawRotation(0, Rotation.Yaw, 0);
        const FVector ForwardDirection = FRotationMatrix(YawRotation).GetUnitAxis(EAxis::X);
        const FVector RightDirection = FRotationMatrix(YawRotation).GetUnitAxis(EAxis::Y);
        AddMovementInput(ForwardDirection, MovementVector.Y);
        AddMovementInput(RightDirection, MovementVector.X);
    }
}

    }

    物理引擎

    物理子步

    UE5 使用可变帧率，但物理引擎在小固定时间步长下效果最好，这会对物理引擎造成困难。设置子步时，会先获取总帧时，并将其划分为子步，然后逐帧多次 ticked 模拟。采样的子步数取决于设置的最大部分时间差量。

    使用 Sub-Stepping 可以让物理模拟更为精确和稳定，但会带来高性能开销。在“编辑-项目设置-物理-帧率”中找到“分布”选项

    其中几个重要选项解释如下

    属性 描述
    分步 在当前项目中启用或停用分步
    分步异步 是否对异步物理模拟设置子步
    最大分步时间差量 一个子步允许花费的最长时间（秒）。例如整步 0.05 秒，最大分步时间差量 0.025 秒，则分为两个子步。若整步时间较长，则根据最大分步数均匀划分，避免过度细分。注意此属性会限制物理步长花费的时间
    最大分步 一个整步可划分出的最大子步数

    注意设置子步后，碰撞回调函数的行为。虚幻引擎以一个单独的物理线程执行物理子步。为了获得最佳性能，将延迟碰撞回调函数，直到完成最后一个子步。因此一次碰撞可能产生多个回调。例如 A, B 碰撞可能得到两者重叠和不再重叠的回调。这些回调保存在队列中，按顺序执行。

    单线程

    默认情况下，UE5 使用多线程模拟物理。如果需要使用单线程模拟，只需要找到项目的 Binaries 目录，命令行运行 exe 文件。例如

    ChaosTestEditor.exe -SingleThreadedPhysics

    就会打开项目，此时运行游戏就会使用单线程模拟物理。
