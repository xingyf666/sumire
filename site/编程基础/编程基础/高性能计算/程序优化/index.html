    程序优化

    性能优化

    常量折叠

    尽量避免代码复杂化，避免使用会造成 new/delete 的容器，从而方便编译器优化。例如

    • 堆上的容器：vector, map, set, String, function, any, unique_ptr, shared_ptr, weak_ptr
    • 栈上的容器：array, bitset, string_view, pair, tuple, optional, variant

    存在栈上的容器利于优化。另外，即使使用栈上的容器，如果代码过多，编译器也会放弃优化，避免编译时间过长

    int func() {
  std::array<int, 100> arr{};
  for (int i = 1; i <= 100; i++) {
    arr[i - 1] = i;
  }
  int ret = 0;
  for (int i = 1; i <= 100; i++) {
    ret += arr[i - 1];
  }
  return ret;
}

    如果希望能强制优化，可以使用 constexpr 迫使编译器进行常量折叠

    constexpr int func() {
  std::array<int, 100> arr{};
  for (int i = 1; i <= 100; i++) {
    arr[i - 1] = i;
  }
  int ret = 0;
  for (int i = 1; i <= 100; i++) {
    ret += arr[i - 1];
  }
  return ret;
}

    函数调用

    如果一个函数调用外部函数，则无法编译优化

    int other(int a);

int func() {
    return other(233);
}

    在 gcc 编译器中使用 @PLT (Procedure Linkage Table) 函数链接表标记函数入口点。链接器将会查找其它 .o 文件中定义的函数符号，如果存在，则会将 @PLT 替换为函数地址；在 windows 中使用 IAT (Import Address Table) 和延迟加载来处理链接，因此不需要 @PLT 标记。

    但是如果外部函数的定义与调用函数在同一个文件中，则编译器可以将对应的函数内联化

    int other(int a) {
    return a;
}

int func() {
    return other(233);
}

// 内联化
int func() {
    return 233;
}

    指针别名

    考虑如下代码

    void func(int *a, int *b, int *c)
{
    *c = *a;
    *c = *b;
}

    似乎后一条语句可以覆盖前一条，但是这里可能存在别名问题

    int a = 10;
int b = 20;
func(&a, &b, &b);

    如果编译器优化掉第一条指令，显然会出错，因此不会优化。

    如果希望提醒编译器不会出现相同的指针，可以使用 __restrict 关键字

    void func(const int *a, const int *b, int *__restrict c)
{
    *c = *a;
    *c = *b;
}

    由于只有写入才会出现指针别名问题，因此只需要为非常量指针添加修饰。

    禁止优化

    使用 volatile 关键字禁止编译器优化，主要用来进行性能测试。例如

    for (volatile int i = 0; i < 100; i++) {}

    由于 i 的写入不能被优化，因此空循环体将会被保留。

    数组优化

    合并写入

    对于内存的连续写入可以被合并，而跳跃写入无法优化。例如

    void func(int *a)
{
    a[0] = 123;
    a[1] = 456;
}

void func(int *a)
{
    a[0] = 123;
    a[2] = 456;
}

    其中前者写入两个 32 位数据会被合并为写入一个 64 位数据，后者则不会。类似的，四个 int32 可以优化为一个 __m128，八个 int32 可以优化为一个 __m256 。

    数组清零

    对于数组的清零操作会被自动优化为 memset 实现

    void func(int *a) {
  for (int i = 0; i < 1024; i++) {
    a[i] = 0;
  }
}

    类似的，数组拷贝也会被优化为 memcpy 实现。

    数组填充

    循环填充连续数组将会被优化为 SIMD 指令

    void func(int *a) {
  for (int i = 0; i < 1024; i++) {
    a[i] = i;
  }
}

// 优化后的伪代码
void func(int *a) {
  __m128i curr = {0, 1, 2, 3};
  __m128i delta = {4, 4, 4, 4};
  for (int i = 0; i < 1024; i += 4) {
    a [i : i + 4] = curr;
    curr += delta;
  }
}

    优化后一次处理 4 个 int 的填入。对于填充长度不为 4 的倍数，编译器将会自动考虑边界问题，对于不能整除的部分生成对于的分支计算。

    如果长度总是 4 的倍数，可以手动提醒编译器

    void func(int *a, int n) {
  n = n / 4 * 4;
  for (int i = 0; i < n; i++) {
    a[i] = i;
  }
}

    编译器会发现 \(4\mid n\)，从而不会生成多余的边界判断分支。

    字节对齐

    在 C++20 中引入了标准化的 std::assume_aligned，用来表明指针是字节对齐的

    void func(int *a, int n) {
  n = n / 4 * 4;
  a = std::assume_aligned<16>(a);
  for (int i = 0; i < n; i++) {
    a[i] = i;
  }
}

    循环优化

    循环矢量化

    对于下面的循环代码

    void func(float *a, float *b) {
  for (int i = 0; i < 1024; i++) {
    a[i] = b[i] + 1;
  }
}

    由于编译器需要考虑指针是否重叠，因此将会生成两份代码，一份是 SIMD 版本，一份是传统标量版本。如果指针没有重叠，则会调用 SIMD 版本；否则使用标量版本。

    但是如果传入的指针数量过多，则编译器将不会再优化。为了方便编译器优化，还是应当添加 __restrict 关键字

    void func(float *__restrict a, float *__restrict b) {
  for (int i = 0; i < 1024; i++) {
    a[i] = b[i] + 1;
  }
}

    除此之外，还可以使用 OpenMP 指令强制矢量化

    void func(float *a, float *b) {
#pragma omp simd
  for (int i = 0; i < 1024; i++) {
    a[i] = b[i] + 1;
  }
}

    分离条件

    如果循环体内部有简单的条件分支

    void func(float *__restrict a, float *__restrict b, bool is_mul) {
  for (int i = 0; i < 1024; i++) {
    if (is_mul)
      a[i] = a[i] * b[i];
    else
      a[i] = a[i] + b[i];
  }
}

    则编译器会将分支提取到外面，便于矢量化

    void func(float *__restrict a, float *__restrict b, bool is_mul) {
  if (is_mul) {
    for (int i = 0; i < 1024; i++)
      a[i] = a[i] * b[i];
  } else {
    for (int i = 0; i < 1024; i++)
      a[i] = a[i] + b[i];
  }
}

    只要条件判断是固定的，不会随循环改变，就可以由编译器自动优化。

    提取不变量

    当循环过程中存在不变量时，编译器可能将其提取出来。例如

    void func(float *__restrict a, float *__restrict b, float dt) {
    // 不会优化
    for (int i = 0; i < 1024; i++) {
        a[i] = a[i] + b[i] * dt * dt;
    }

    // 提取不变量
    // double t = dt * dt;
    for (int i = 0; i < 1024; i++) {
        a[i] = a[i] + b[i] * (dt * dt);
    }
}

    由于浮点数乘法不具有结合性，因此需要显式指定相乘的顺序，编译器才能将其提取处理。

    优化失败

    通常非顺序访问数组都会对编译器优化产生障碍

    • 随机访问
    • 跳跃访问

    循环展开

    当循环体相对于循环结束条件较为复杂时，结束条件判断将成为性能瓶颈。例如

    void func(float *a) {
  for (int i = 0; i < 1024; i++)
    a[i] = 1;
}

    此时大部分时间都耗费在循环条件判断中，解决方案是将循环展开

    void func(float *a) {
  for (int i = 0; i < 256; i++) {
    a[i + 0] = 1;
    a[i + 1] = 1;
    a[i + 2] = 1;
    a[i + 3] = 1;
  }
}

    要使用循环展开，建议通过编译器提供的语法

    void func(float *a) {
#pragme unroll 4
  for (int i = 0; i < 1024; i++)
    a[i] = 1;
}

    优化思路

    • 首先，几乎不应该手动进行循环展开，这会导致编译器的分析复杂化，并且生成的代码可能会更慢。但是，可以使用编译器的 #pragma 强制循环展开。对于循环流水线，最好将这种优化留给编译器。
    • 手动矢量化的唯一方法是使用矢量化编译指示，例如 #pragma omp simd，不过仍然不建议使用：
      • 如果编译器没有对循环矢量化，则存在特定原因
    • 编译器很少进行循环交换，对于热代码上的循环，绝对应该手动完成，例如交换矩阵和图像处理的循环代码
    • 如果循环不可矢量化，则优化可以将循环拆分成可矢量化和不可矢量化的部分。但是仍然需要检测在不同编译器下的运行效果
    • 通常将循环合并会有积极影响，但是存在例外：如果编译器对其中某个循环进行矢量化，而合并后取消了矢量化，则影响可能是负面的

    结构体优化

    辅助对齐

    考虑 8 字节的结构体操作

  float x;
  float y;
  // float z;

  float z;

    内存布局

    AOS (Array of Struct) 指单个对象的属性紧挨着存放；SOA (Struct of Array) 指属性分离存储在多个数组中。AOS 必须对齐到 2 的幂才高效，而 SOA 则不需要。

    使用 AOS 和 SOA 两种布局方式和它们相互结合的布局 AOSOA，可以得到下面 3 种等价的代码

    struct AOS_Vec {
  float x;
  float y;
  float z;
};

AOS_Vec a[1024];

void AOS_func() {
  for (int i = 0; i < 1024; i++)
    a[i].x *= a[i].y;
}

struct SOA_Vec {
  float x[1024];
  float y[1024];
  float z[1024];
};

SOA_Vec b;

void SOA_func() {
  for (int i = 0; i < 1024; i++)
    b.x[i] *= b.y[i];
}

struct AOSOA_Vec {
  float x[4];
  float y[4];
  float z[4];
};

AOSOA_Vec c[1024 / 4];

void AOSOA_func() {
  for (int i = 0; i < 1024 / 4; i++) {
    for (int j = 0; j < 4; j++) {
      c[i].x[j] *= c[i].y[j];
    }
  }
}

    数学优化

    乘除法

    编译器会自动将除法转换为更高效的乘法。例如

    a /= 2; -> a *= 0.5;

    然而，虽然乘法更快，但是编译器通常无法将除法提取为倒数

    void func(float *a, float b) {
  for (int i = 0; i < 1024; i++)
    a[i] /= b;
}

    由于编译器需要考虑 b 是否为零，因此不会优化。可以手动提取

    void func(float *a, float b) {
  float inv_b = 1 / b;
  for (int i = 0; i < 1024; i++)
    a[i] *= inv_b;
}

    在 GCC 中可以开启 -ffast-math 选项，允许编译器执行非标准、激进的浮点优化，以便提高性能。在 MSVC 中可以开启 /fp:fast 选项，启用快速浮点运算优化。

    数学函数

    应当使用 std:: 中的数学函数，其中提供了对不同数据类型的重载版本，实现更为高效。

    开启 fast-math 选项后，对于某些数学函数，编译器将会采取更激进的优化方案。例如

    void func(float *a) {
  for (int i = 0; i < 1024; i++)
    a[i] = std::sqrt(a[i]);
}

    正常情况下，编译器需要考虑 a[i] 是否非负，因此不会矢量化；而开启 fast-math 后，将会忽略非负检测。

    • 对于浮点运算，加法和乘法速度相同
    • L1/2/3 read 和 Main RAM Read 的耗时是读取一个缓存行（64 字节）的耗时。因此从 Main RAM 中读取一次 float 耗时为 100~150/64*4 个 CPU 周期

    多级缓存

    CPU 内部具有缓存结构 $$ CPU\to 寄存器\to L_1 缓存 \to L_2缓存 \to L_3 共享缓存 \to 局部内存 $$ 当 CPU 访问某个地址时，会逐级查找缓存中是否有对应的数组。当访问数据量较小时，就可以预先将数据加载到更高效的缓存中，避免内存读写的延迟。

    缓存读取

    当 CPU 读取一个地址时，缓存会自动查找与该地址匹配的条目。如果找不到，就会向主内存发送请求，每次读取一个缓存行 cacheline，在 x86 架构中每个缓存行储存 64 字节数据。因此将数据结构对齐到缓存行可以更高效地利用每次读取。

    缓存写入

    当 CPU 写入一个地址时，缓存会自动查找与该地址匹配的条目。如果找不到，就会创建一个新的缓存行储存 CPU 写入的数据，标记为脏数据 dirty 。当缓存中保存的数据过多，就会将不常用的缓存行移除，这称为失效。如果对应的缓存行是脏数据，则会向主内存发送写入请求，写入完成后才会移除。

    如果有多级缓存，则上一级缓存的失效数据会保存在下一级缓存。

    考虑一个简单的循环迭代 $$ a^{[k+1]} = Aa^{[k]},\quad A = \begin{bmatrix} 0 & 1\ 1 & 0 & 1\ & & & \ddots\ & & & 1 & 0 \end{bmatrix} $$ 对比直接循环迭代和一次计算两步迭代的展开循环的耗时

constexpr std::size_t n = 1 << 26;
constexpr std::size_t steps = 32;

std::vector<float> a(n);
std::vector<float> b(n);

void jacobi() {
#pragma omp parallel for
  for (std::size_t i = 0; i < n; i++) {
    a[i] = std::sin(i * 0.1f);
  }

  TICK(iter);
  for (int step = 0; step < steps; step++) {
#pragma omp parallel for
    for (std::size_t i = 1; i < n - 1; i++) {
      b[i] = (a[i - 1] + a[i + 1]) * 0.5f;
    }
    std::swap(a, b);
  }
  TOCK(iter);

  float loss = 0;
#pragma omp parallel for reduction(+ : loss)
  for (std::size_t i = 1; i < n - 1; i++) {
    loss += std::pow(a[i - 1] + a[i + 1] - a[i] * 2, 2);
  }
  loss = std::sqrt(loss);
  std::cout << "loss: " << loss << std::endl;
}

void jacobi_local() {
#pragma omp parallel for
  for (std::size_t i = 0; i < n; i++) {
    a[i] = std::sin(i * 0.1f);
  }

  TICK(iter);
  for (int step = 0; step < steps / 16; step++) {
#pragma omp parallel for
    for (std::size_t ibase = 16; ibase < n - 16; ibase += 32) {
      float ta[32 + 16 * 2], tb[32 + 16 * 2];
      for (int i = -16; i < 32 + 16; i++) {
        ta[16 + i] = a[ibase + i];
      }
      for (int s = 1; s < 16; s += 2) {
        for (int i = -16 + s; i < 32 + 16 - s; i++) {
          tb[16 + i] = (ta[16 + i - 1] + ta[16 + i + 1]) * 0.5f;
        }
        for (int i = -16 + s + 1; i < 32 + 16 - s - 1; i++) {
          ta[16 + i] = (tb[16 + i - 1] + tb[16 + i + 1]) * 0.5f;
        }
      }
      for (int i = 0; i < 32; i++) {
        b[ibase + i] = tb[16 + i];
      }
    }
    std::swap(a, b);
  }
  TOCK(iter);

  float loss = 0;
#pragma omp parallel for reduction(+ : loss)
  for (std::size_t i = 1; i < n - 1; i++) {
    loss += std::pow(a[i - 1] + a[i + 1] - a[i] * 2, 2);
  }
  loss = std::sqrt(loss);
  std::cout << "loss: " << loss << std::endl;
}

    多维数组

    二级指针

    内存是一维的，即使声明二维数组，也会被扁平化后储存在内存中。对于 float a[3][4]，编译器会先将其变为一维数组 float a[3*4]，然后将 a[i][j] 翻译为 a[i*4+j]，这意味着我们仍然在处理一维数组。然而，二级指针在结构上就不再是一维的

    float **a = new float *[n];
for (int i = 0; i < n; i++)
  a[i] = new float[n];

    这里执行了 n 次内存分配，内存都是分散的，每次访问都需要解开两层指针，非常低效。

    不要使用 std::vector<std::vector<float>> 分配二维数组，因为 vector 本身是 24 字节，更占用内存。而应该直接使用一维数组或者 std::arrry<std::array<float, n>, m> 封装。

    访问越界

    经常遇到需要访问一个元素周围的几个元素，这就可能存在访问越界的情况，一般有几种解决方案

    1. 使用 std::max, std::min 控制索引范围，但是会导致难以矢量化
    2. 使用 (i + n) % n 使索引回绕，但是更低效并且同样难以矢量化
    3. 修改遍历区间，为边界留出空间
    4. 分配大于原始数组的扩展数组，为边界留出 ghost cell

    多维数组

    利用模板实现多维数组类型，并提供 ghost cell 边界。数组元素 a(i, j) 被扁平化为 a[i + j * N]，因此这是一个列主序的类型。

    #pragma once

#include <array>
#include <cstddef>
#include <cstdint>
#include <stdexcept>
#include <type_traits>
#include <vector>

template <std::size_t N, class T, std::size_t LoBound = 0,
          std::size_t HiBound = LoBound>
class arrayxd {
  static_assert(N > 0, "N cannot be 0");
  static_assert(std::is_same_v<std::remove_reference_t<std::remove_cv_t<T>>, T>,
                "T cannot be cvref");

  using Dim = std::array<std::intptr_t, N>;
  using Shape = std::array<std::size_t, N>;

  std::vector<T> m_arr;
  Shape m_shape{};

  constexpr static std::size_t _calc_size(const Shape &shape) noexcept {
    std::size_t size = shape[0] + (LoBound + HiBound);
    for (std::size_t i = 1; i < N; i++) {
      size *= shape[i] + (LoBound + HiBound);
    }
    return size;
  }

public:
  arrayxd() = default;
  arrayxd(const arrayxd &) = default;
  arrayxd(arrayxd &&) = default;
  arrayxd &operator=(const arrayxd &) = default;
  arrayxd &operator=(arrayxd &&) = default;
  ~arrayxd() = default;

  explicit arrayxd(const Shape &shape)
      : m_arr(_calc_size(shape)), m_shape(shape) {}

  explicit arrayxd(const Shape &shape, const T &value)
      : m_arr(_calc_size(shape)), m_shape(shape) {}

  template <class... Ts>
    requires(sizeof...(Ts) == N && (std::is_integral_v<Ts> && ...))
  explicit arrayxd(const Ts &...ts) : arrayxd(Shape{ts...}) {}

  void reshape(const Shape &shape) {
    std::size_t size = _calc_size(shape);
    m_shape = shape;
    m_arr.clear();
    m_arr.resize(size);
  }

  void reshape(const Shape &shape, const T &value) {
    std::size_t size = _calc_size(shape);
    m_shape = shape;
    m_arr.clear();
    m_arr.resize(size, value);
  }

  void compress() {
    // release memory unused
    // capacity -> size
    m_arr.shrink_to_fit();
  }

  template <class... Ts>
    requires(sizeof...(Ts) == N && (std::is_integral_v<Ts> && ...))
  void reshape(const Ts &...ts) {
    this->reshape(Shape{ts...});
  }

  constexpr Shape shape() const noexcept { return m_shape; }

  constexpr std::size_t shape(std::size_t i) const noexcept {
    return m_shape[i];
  }

  constexpr std::size_t index(const Dim &dim) const noexcept {
    std::size_t offset{dim[0] + LoBound};
    std::size_t term = 1;
    for (std::size_t i = 1; i < N; i++) {
      term *= m_shape[i - 1] + (LoBound + HiBound);
      offset += term * std::size_t{dim[i] + LoBound};
    }
    return offset;
  }

  constexpr std::size_t safe_index(const Dim &dim) const noexcept {
    for (std::size_t i = 0; i < N; i++) {
      if (dim[i] < -std::intptr_t{LoBound} || dim[i] >= m_shape[i] + HiBound)
        throw std::out_of_range("arrayxd at");
    }
    return this->index(dim);
  }

  constexpr T *data() noexcept { return m_arr.data(); }

  constexpr const T *data() const noexcept { return m_arr.data(); }

  constexpr T &operator()(const Dim &dim) noexcept {
    return this->data()[this->index(dim)];
  }

  constexpr const T &operator()(const Dim &dim) const noexcept {
    return this->data()[this->index(dim)];
  }

  template <class... Ts>
    requires(sizeof...(Ts) == N && (std::is_integral_v<Ts> && ...))
  constexpr T &operator()(const Ts &...ts) noexcept {
    return operator()(Dim{ts...});
  }

  template <class... Ts>
    requires(sizeof...(Ts) == N && (std::is_integral_v<Ts> && ...))
  constexpr const T &operator()(const Ts &...ts) const noexcept {
    return operator()(Dim{ts...});
  }

  constexpr T &operator[](const Dim &dim) noexcept {
    return operator()(this->index(dim));
  }

  constexpr const T &operator[](const Dim &dim) const noexcept {
    return operator()(this->index(dim));
  }

  T &at(const Dim &dim) { return this->data()[this->safe_index(dim)]; }

  const T &at(const Dim &dim) const {
    return this->data()[this->safe_index(dim)];
  }

  template <class... Ts>
    requires(sizeof...(Ts) == N && (std::is_integral_v<Ts> && ...))
  T &at(const Ts &...ts) {
    return this->at(Dim{ts...});
  }

  template <class... Ts>
    requires(sizeof...(Ts) == N && (std::is_integral_v<Ts> && ...))
  const T &at(const Ts &...ts) const {
    return this->at(Dim{ts...});
  }
};

    注意到维数类型使用 std::intptr_t，这是用来保存指针的类型。使用它是为了能够与 int 类型兼容，它可以使用例如 Dim{ts...} 的初始化列表。

    预取 + 直写

    利用预取机制配合内存直写，同时注意到我们将 offset 循环了两遍，这是为了利用 CPU 指令并行（ILP）：当对不同的寄存器执行指令时，可以启动并行计算。

    例如下面遍历 res 的 4 个不同的元素，如果将 offset 循环放在外面，可以和前面的初始化合并，然而由于内循环将始终访问同一个 res[offset]，就浪费了寄存器资源。反之，每个元素存放在不同寄存器中，互不干扰，这 4 个元素的计算就可以实现并行（由于寄存器数量有限，因此要控制元素数量不能太多）。

    void BM_y_blur_tiled_only_x_prefetch_streamed_merged(benchmark::State &state) {
  constexpr int nblur = 8;
  arrayxd<2, float, nblur> a(nx, ny);
  arrayxd<2, float> b(nx, ny);

  for (auto _ : state) {
    constexpr int blockSize = 32;
#pragma omp parallel for collapse(2)
    for (int xBase = 0; xBase < nx; xBase += blockSize) {
      for (int y = 0; y < ny; y++) {
        for (int x = xBase; x < xBase + blockSize; x += 16) {
          // 预取机制
          _mm_prefetch((char *)&a(x, y + nblur), _MM_HINT_T0);
          // 使用 xmm 寄存器
          __m128 res[4];
          for (int offset = 0; offset < 4; offset++) {
            res[offset] = _mm_setzero_ps();
          }
          for (int t = -nblur; t <= nblur; t++) {
            // 启用指令级并行
            for (int offset = 0; offset < 4; offset++) {
              res[offset] = _mm_add_ps(res[offset],
                                       _mm_load_ps(&a(x + offset * 4, y + t)));
            }
          }
          // 内存直写，放在一个循环里，保持数据热度，避免其它指令过多导致直写被放弃
          for (int offset = 0; offset < 4; offset++) {
            _mm_stream_ps(&b(x + offset * 4, y), res[offset]);
          }
        }
      }
    }
    benchmark::DoNotOptimize(a);
  }
}

    注意到访问序在遍历每行时，都会跨越矩阵行数对应的内存；换行时，会跨越从最后一列到第一列的距离，缓存无法容纳，这就导致无法预取。改为使用分块序后，每次只遍历行的一部分，此时换行只会跨越 1/3 的列数。只要块足够小，使缓存能够装下每一块的数据，那么圈出的部分由于内存连续而被预取，当换行时就可以直接利用预取的数据，减少了内存访问开销。

    莫顿码

    莫顿码 morton code 是一种分形编码技术，它将每一维的坐标记为二进制，然后将它们交错打包。例如 $$ x = 010 = x_2x_1x_0,\quad y=001=y_2y_1y_0,\quad mdec(x, y) = y_2x_2y_1x_1y_0x_0= 000110 $$ 这种方式编码得到的访问顺序就是下图中的 Z 型分形曲线。

    由于分形特性和每级缓存大小的倍数关系，这种分块方式能够自适应任何大小的多级缓存，从而有利于矢量化。

    void BM_transpose_tiled_morton(benchmark::State &state) {
  for (auto _ : state) {
    constexpr int blockSize = 64;
#pragma omp parallel for collapse(2)
    for (int mCode = 0; mCode < (ny / blockSize) * (nx / blockSize); mCode++) {
      auto [xBase, yBase] = morton2d::decode(mCode);
      xBase *= blockSize;
      yBase *= blockSize;
      for (int y = yBase; y < yBase + blockSize; y++) {
        for (int x = xBase; x < xBase + blockSize; x++) {
          b(x, y) = a(y, x);
        }
      }
    }
    benchmark::DoNotOptimize(b);
  }
}

    读写分离

    由于使用 [] 访问元素时，如果对于元素不存在就会创建元素，为了实用性考虑，如果碰到不存在的元素最好返回空值，因此我们将读写操作分离开。并且使用 unordered_map 保存，提高访问性能。

    struct Sparse_RW {
  struct Hash {
    std::size_t operator()(const std::tuple<int, int> &key) const {
      auto &[x, y] = key;
      return (x * 2718281828) ^ (y * 3141592653);
    }
  };

  std::unordered_map<std::tuple<int, int>, float, Hash> m_data;
  // std::map<std::tuple<int, int>, float> m_data;

  float read(int x, int y) const {
    auto it = m_data.find(std::make_tuple(x, y));
    if (it != m_data.end())
      return it->second;
    return 0;
  }

  void write(int x, int y, float value) {
    auto it = m_data.find(std::make_tuple(x, y));
    if (it != m_data.end())
      it->second = value;
  }

  void create(int x, int y, float value) {
    m_data.emplace(std::make_tuple(x, y), value);
  }

  template <class Func> void foreach (Func &&func) {
    for (auto &[key, value] : m_data) {
      auto &[x, y] = key;
      func(x, y);
    }
  }
};

    struct Sparse_Vec_Compressed {
  std::vector<std::tuple<int, float>> m_data;
  int m_row_offset[N]{};

  void createRow(int x, const std::vector<std::tuple<int, float>> &values) {
    m_row_offset[x] = m_data.size();
    for (auto &value : values)
      m_data.push_back(value);
  }

  template <class Func> void foreach (Func &&func) {
    for (int x = 0; x < N; x++) {
      int row_end = x == N - 1 ? m_data.size() : m_row_offset[x + 1];
      for (int offset = m_row_offset[x]; offset < row_end; offset++) {
        auto &[y, value] = m_data[offset];
        func(x, y, value);
      }
    }
  }
};

    运动轨迹

    考虑从原点以固定矢量运动的轨迹，使用一个网格来表示运动空间，标记运动轨迹对应的位置。标准的网格结构为

struct Grid {

    压缩网格

    注意到一个 char 类型有 1 字节 256 种可能，但是我们只用到其中的 0/1 两种情况。于是我们将每个 char 的 8 位二进制中的每一位来表示两种可能，这样就可以将列数压缩 8 倍

    struct Grid_8 {
  char m_data[N][N / 8];

  bool read(int x, int y) const {
    return m_data[x][y / 8] & (1 << (y % 8)) ? true : false;
  }

  void write(int x, int y, int value) {
    if (value)
      m_data[x][y / 8] |= 1 << (y % 8);
    else
      m_data[x][y / 8] &= ~(1 << (y % 8));
  }
};

    稀疏化

    使用 unordered_map 表示稀疏网格，即使用前面稀疏矩阵的形式保存

    使用这种稀疏形式，坐标可以是负数，因此更为灵活。

    struct Grid_Map {
  struct Hash {
    std::size_t operator()(const std::tuple<int, int> &key) const {
      auto &[x, y] = key;
      return (x * 2718281828) ^ (y * 3141592653);
    }
  };

  std::unordered_map<std::tuple<int, int>, char, Hash> m_data;

  char read(int x, int y) const {

    if (it != m_data.end())
      return it->second;
    return 0;

  void write(int x, int y, char value) {

    if (it != m_data.end())
      it->second = value;
  }

  template <class Func> void foreach (Func &&func) {
    for (auto &[key, value] : m_data) {
      auto &[x, y] = key;
      func(x, y, value);
    }
  }
};

    循环分块

    使用 unordered_map 虽然能够最小化内存占用，但是却导致不连续的内存分配。当我们使用到网格中的某个元素时，通常会用到其周围的元素。因此我们可以将网格分块，在底层保证局域连续性，在上层使用 unordered_map 管理。

    struct Grid_Map_B {
  struct Hash {
    std::size_t operator()(const std::tuple<int, int> &key) const {
      auto &[x, y] = key;
      return (x * 2718281828) ^ (y * 3141592653);
    }
  };

  static constexpr int B = 16;

  struct Block {
    char m_block[B][B];
  };

  std::unordered_map<std::tuple<int, int>, Block, Hash> m_data;

  char read(int x, int y) const {
    auto it = m_data.find(std::make_tuple(x / B, y / B));
    if (it != m_data.end())
      return it->second.m_block[x % B][y % B];
    return 0;
  }

  void write(int x, int y, char value) {
    Block &block = m_data[std::make_tuple(x / B, y / B)];
    block.m_block[x % B][y % B] = value;
  }

  template <class Func> void foreach (Func &&func) {
    for (auto &[key, block] : m_data) {
      auto &[xb, yb] = key;
      for (int dx = 0; dx < B; dx++) {
        for (int dy = 0; dy < B; dy++) {
          func(xb * B + dx, yb * B + dy, block.m_block[dx][dy]);
        }
      }
    }
  }
};

    负数计算

    取余问题

    前面提到稀疏化可以使用负数坐标，但是直接用 % 计算分块坐标将会得到负索引。为了解决取余问题

    • 一种方案是使用 (a + b) % b 形式，但如果 a < -b 还是会出错
    • 正确的写法是 (a % b + b) % b
    • 若 \(b=2^n\)，则高效的解决方案是 a % b = a & (b - 1)，相当截取 a 的后 n 位

    对负数取余时，能够确保得到正数 $$ \begin{matrix} & 4 & 3 & 2 & 1 & 0 & -1 & -2 & -3 & -4\ \% 4 & 0 & 3 & 2 & 1 & 0 & -1 & -2 & -3 & -1\ \& 3 & 0 & 3 & 2 & 1 & 0 & 3 & 2 & 1 & 0 \end{matrix} $$ 可以看到使用 &3 得到了预期中的结果。

    除法问题

    除以 2 的幂可以转换为位移运算，例如 a / 4 = a >> 2，并且使用位移可以解决负数除法问题 $$ \begin{matrix} & 4 & 3 & 2 & 1 & 0 & -1 & -2 & -3 & -4\ / 4 & 1 & 0 & 0 & 0 & 0 & 0 & 0 & 0 & -1\

    2 & 1 & 0 & 0 & 0 & 0 & -1 & -1 & -1 & -1\ \end{matrix} $$ 注意到默认的除法会向上取整，这会导致 -3~3 都落在 0 对应的块中，显然不符合要求。

    加法运算

    如果保证 a 和 b 满足 a & b = 0，那么 a + b = a | b，后者更为高效。

    嵌套分块

    使用位运算替换所有除法、取余和加法操作，并且使用嵌套的分块

    1. 顶层使用二维指针块，只有存在元素的块会申请内存，确保稀疏性
    2. 底层使用 256 x 256 块，确保局域连续性

    还可以在最外层使用 unordered_map嵌套，实现多层解耦。

    双层嵌套的网格结构如下

    struct Grid_Bshift {
  static constexpr int Bshift = 8;
  static constexpr int B = 1 << Bshift;
  static constexpr int Bmask = B - 1;

  static constexpr int B1shift = 11;
  static constexpr int B1 = 1 << B1shift;
  static constexpr int B1mask = B1 - 1;

  struct Block {
    char m_block[B][B];
  };

  std::unique_ptr<Block> m_data[B1][B1];

  char read(int x, int y) const {
    auto &block = m_data[(x >> Bshift) & B1mask][(y >> Bshift) & B1mask];
    if (!block)
      return 0;
    return block->m_block[x & Bmask][y & Bmask];
  }

  void write(int x, int y, char value) {
    auto &block = m_data[(x >> Bshift) & B1mask][(y >> Bshift) & B1mask];
    if (!block)
      block = std::make_unique<Block>();
    block->m_block[x & Bmask][y & Bmask] = value;
  }

  template <class Func> void foreach (const Func &func) {
    for (int x1 = 0; x1 < B1; x1++) {
      for (int y1 = 0; y1 < B1; y1++) {
        auto &block = m_data[x1 & B1mask][y1 & B1mask];
        if (!block)
          continue;
        int xb = x1 << B1shift;
        int yb = y1 << B1shift;
        for (int dx = 0; dx < B; dx++) {
          for (int dy = 0; dy < B; dy++) {
            func(xb | dx, yb | dy, block->m_block[dx][dy]);
          }
        }
      }
    }
  }
};

    浮点精度

    计算机使用科学计数法保存浮点数，格式为 $$ \pm f \times 2^e $$ 其中 f 是底数，e 是指数。单精度 float 具有 1 个符号位、8 个指数位和 23 个底数位；双精度 double 具有 1 个符号位、11 个指数位和 52 个底数位。这意味着，单精度表示精度为 23 位二进制，至多表示 7 位数。例如 $$ 1234000.0 + 0.01 = 1234000.0 $$ 这是因为 \(1234000.1\) 需要 9 位有效数字。

    定点数

    定点数是指固定位系数，然后将有效位数储存为整型。例如指定位系数为 \(100\)，则 \(123.4\) 表示为 \(12340\)，只需要将定点数除以 100 就得到浮点数。定点数的表示范围比浮点数小，但是由于使用整型和固定位数存储，因此精度均匀。
