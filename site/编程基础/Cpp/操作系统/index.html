    操作系统

    概述

    并发

    单核 CPU 同一时刻只能执行一个程序，各个程序只能并发地执行。多核 CPU 同一时刻可以同时执行多个程序。

    • 并发：多个事件在同一时刻内发生，操作系统的并发性是指同时存在多个运行的程序。在宏观上是一起发生，但在微观上是交替执行的
    • 并行：同一时间内多个时间同时发生，宏观和微观上都是发生的

    共享

    共享即资源共享，是指系统中的资源被多个并发进程使用。

    • 同时共享方式：系统中的某些资源，一个时间段内由多个进程“同时”对它们访问
    • 互斥共享方式：系统中的某些资源，一个时间段内只允许一个进程使用

    虚拟

    虚拟是指把物理上的实体变为逻辑上的对应物。

    异步

    异步是指在多道程序环境下，允许多个程序并发执行，但由于资源优先，进程执行不是一贯到底的，而是走走停停，以不可预知的速度推进。

    运行机制

    内核态和用户态

    CPU 有两种状态

    • 内核态：此时运行的是内核程序，此时可以执行特权指令
    • 用户态：此时运行的是应用程序，此时只能执行非特权指令

    CPU 中有一个寄存器，用 1 表示内核态，0 表示用户态。

    进程与线程

    进程

    概念

    进程用于实现操作系统的并发性和共享性，以便更好地描述和控制程序的并发执行。进程实体由三部分组成：程序段、数据段和 PCB (进程控制块) 组成。PCB 是进程存在的唯一标志，是给操作系统用的；而程序段和数据段是给进程自己使用的。

    通信

    进程是分配系统资源的单位（包括内存地址空间），因此各进程拥有的内存地址空间相互独立。为了保证安全，一个进程不能直接访问另一个进程的地址空间。

    线程

    概念

    有些进程可能需要“同时”做很多事，而传统的进程只能串行地执行一系列程序。引入线程用于增加并发度。线程是一个基本地 CPU 执行单元，也是程序执行流地最小单位。引入线程后，可以在进程内的各线程之间并发。

    进程 线程
    资源 资源分配的基本单位 资源调度的基本单位
    并发性 进程间并发 进程内并发
    系统开销 开销大 开销小
    地址空间和资源 每个进程之间独立的地址空间和资源 共享地址空间和资源

    用户级线程

    由应用程序通过线程库实现，所有线程的管理工作都由应用程序负责（包括线程切换）。在用户看来的多个线程，在操作系统看来并不存在。

    • 优点：在用户空间可以完成线程切换，不需要切换到核心态，开销小，效率高
    • 缺点：当用户级线程被阻塞后，整个进程都会被阻塞。

    内核级线程

    由操作系统内核完成，因此必须在核心态下进行切换。操作系统会为每个内核级线程建立相应的 TCB (线程控制块)，通过管理线程。

    • 优点：当线程被阻塞后，别的线程可以继续执行，并发能力强
    • 缺点：一个用户进程会占用多个内核级线程，线程切换由操作系统内核完成，因此管理成本高

    内存分配

    连续分配

    内部碎片就是已经被分配出去却不能被利用的内存空间。它处于区域内部或页面内部，而占有这些区域或页面的进程并不使用它，这意味着系统不能利用它。外部碎片是还没有被分配出去，但是由于太小了无法分配给申请内存空间的新进程的内存空闲区域。

    单一连续分配

    这种分配方式下，内存被分为系统区和用户区。内存中只能有一道用户程序，用户程序独占整个用户空间。

    • 优点：实现简单，没有外部碎片，因为只有一个程序分配空间。不一定需要内存保护，也是因为只有一个程序的内存
    • 缺点：只能用于单用户，有内部碎片，储存空间的利用率低，因为所有内存都分配给单个程序，有很多空间用不上

    固定分区分配

    把用户空间分成多个固定大小的分区，每个分区存入一个作业。

    • 优点：实现简单，没有外部碎片
    • 缺点：当用户程序太大时，会降低性能，并且有内部碎片（较大的空间分配给较小的进程）

    动态分区分配

    根据进程的大小动态分配，不预先划分内存分区，而是在进程装入内存时简历分区。

    分配算法 优点 缺点 排序
    首次适应 性能最好，开销小，不需要排序
    最佳适应 优先使用更小的分区，更大的分区保留下来满足大进程的需求 会产生很难利用的碎片，开销大，需要对空闲分区排序 按照空闲分区容量升序
    最坏适应 优先使用更大的分区 不易于大进程，需要排序，因此开销大 按照空闲分区容量降序
    临近适应 首次适应后，每次从上次位置继续查找，开销小 会使高地址的大分区也被用完 按照空闲分区地址升序

    非连续分配

    基本分页储存管理

    我们可以将内存分为多个相等的小分区（页），再按照分区大小把进程拆分成多个部分，各个页面不用连续存放。程序地址与内存地址的关系会映入页表。

    • 页表包括页号和块号。由于顺序排放，所以页号是隐含的，不占用内存空间
    • 页号 = 逻辑地址 / 页面长度
    • 页内偏移量 = 逻辑地址 % 页面长度

    因为页表存放在内存中，为了查询地址访问两次内存会增加开销，所以在高速缓冲存储器中产生了快表（TLB），它是页表的高速缓存，仅保存活跃的页表项，是页表的性能优化补充。

    [!note] 假设有一个 32 位逻辑地址空间的分页系统，每页大小为 4KB，每个表项占用 4 字节，则整个页表需要占用 4MB 连续内存。为了解决可能无法分配足够的连续内存给页表的问题，就诞生了两级页表：将原本连续的页表拆分为离散的多个部分，然后用二级页表（连续内存）保存一级页表的地址。

    基本分段储存管理

    分页是物理单位，可以提高内存的利用率；分段是逻辑单位，便于编程、信息保护和共享、动态增长及动态链接等。按照用户进程自身的逻辑关系划分若干段

    • 代码段（Text Segment）：存放程序指令
    • 数据段（Data Segment）：存放全局变量
    • 堆段（Heap Segment）：动态分配的内存（如 malloc）。
    • 栈段（Stack Segment）：存放函数调用栈、局部变量。

    内存分配时，以段为单位，段内连续，段间可以不连续。逻辑地址由段号和偏移地址定位，操作系统通过段表管理段的内存位置。

    虚拟内存管理

    由于传统存储管理方式（连续分配和非连续分配）都是一次性的，作业必须直接全部装入内存后开始运行。基于局部性原理，在程序装入时，可以将程序中很快会用到的部分装入内存，直接开始执行程序：

    • 当程序访问的信息不在内存中时，由操作系统负责将所需信息从外存调入内存；
    • 若内存空间不够，由操作系统负责将内存中暂时用不到的信息换出外存；

    在操作系统的管理下，在用户看来似乎有一个比实际内存大得多的内存，这就是虚拟内存。

    请求分页管理方式

    请求分页存储管理和基本分页存储管理的主要区别是会调入调出内存。

    页表机制

    页表保存内容包括

    页号 内存块号 状态位 访问字段 修改位 外存地址
    0 A 1 0 0 X
    1 B 1 10 0 Y
    2 C 1 6 1 Z
    其中
    • 状态位：是否已调入内存
    • 访问字段：记录最近被访问的时间
    • 修改位：页面调入内存后是否被修改
    • 外存地址：页面在外存中存放的位置

    在请求分页系统中，若要访问的页面不在内存中时，便会产生缺页中断

    • 若内存中还有空间，则会分配一个空的空闲块
    • 若内存不足，则由页面置换算法淘汰一个页面。若淘汰页面被修改过，则要写回外存，否则直接丢弃

    页面置换算法

    算法 优缺点
    最佳置换算法（OPT） 优先淘汰最长时间不会被访问的 缺页率最好，性能最好，但无法实现
    先进先出置换算法（FIFO） 优先淘汰最先进入内存的页面 实现简单，但性能很差
    最久未使用算法（LRU） 优先淘汰最久未使用的页面 性能很好，但需要硬件支持，算法开销大, 对所有页排序
    时钟置换算法（CLOCK） 优先淘汰访问位为 0 的页面，并将其它页面的访问位更新为 0 实现简单，算法开销小；但未考虑页面是否被修改，修改后置换则还要写入外存，导致额外开销
