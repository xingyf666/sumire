    Shell

    Shell 脚本不仅仅可以在 Linux 系统下运行，还能够在 Windows 下运行来简化操作。在后者系统下，可以双击 .bat 文件直接运行脚本。

    基本介绍

    声明解释器

    对于 Linux 系统，首先应当在第一行声明解释器，如下：

    #! /bin/bash

    表示使用 bash 作为解释器。如果在运行时直接输入解释器就可以省略声明，例如

    /bin/bash test.sh
/bin/php test.php
/bin/python test.py

    要在不同平台上都正确给出解释器，应当使用

    #! /usr/bin/env name

    执行脚本

    一般需要先使脚本具有执行权限，使用

    chmod +x ./test.sh

    然后直接运行脚本

    ./test.sh  # 执行脚本（可以不加.sh后缀）

    在 windows 下，默认目录是 ~，需要 cd .. 到根目录 /，然后

    cd c    # 进入 c 盘
cd d    # 进入 d 盘
cd e    # 进入 e 盘
cd f    # 进入 f 盘

    特殊字符

    shell 中有一些特殊字符，具有特定的含义

    字符 含义
    '' 在单引号中所有的特殊符号，如 $ 和 `` 都没有特殊含义。单引号括起来的都是普通字符，会原样输出
    `` 反引号括起来的内容是系统命令，在 Bash 中会先执行它
    $() 和反引号作用一样，用来引用系统命令(推荐使用)
    () 用于一串命令执行时，()中的命令会在子 Shell 中运行
    {} 用于一串命令执行时，{}中的命令会在当前 Shell 中执行;也可以用于变量变形与替换
    [] 用于变量的测试
    # 在 Shell 脚本中， # 开头的行代表注释
    ; 顺序分隔符

    反引号

    反引号是需要着重注意的特殊符号，因为它通常不怎么出现。使用反引号会将其中执行命令的结果返回，所以可以用作命令参数。例如

    f="hello_finished.txt"
mv $f `${f//_finished.png/}`

    将 "hello_finished.txt" 重命名为 "hello.txt"。

    命令

    输出命令

    有两种可用的方法输出文本

    echo "Hello World."
printf "Hello World.\n"

    上面两行代码的效果相同。其中 printf 提供类似 C 语言的格式化操作。

    调用命令

    shell 脚本的好处在于它可以在内部运行其它程序，例如

    # test.sh

echo 12

# test2.sh

re=`./test`     # 使用反引号接收 test 文件的输出

    多命令

    可以根据命令执行情况依次执行多条命令，只需用下面符号连接

    • & 依次执行每个命令
    • && 前一个命令执行成功后执行下一条
    • || 前一个命令失败后执行下一条

    我们可以用分号连接多条命令，它们也会依次执行。例如

    #! /usr/bin/bash

echo "hello world" & echo "hello world" & echo "hello world"
echo "hello world1" && echo "hello world1" && echo "hello world1"
echo "hello world2" || echo "hello world2" || echo "hello world2"
echo "hello world3" ; echo "hello world3" ; echo "hello world3"

    重定向

    Linux 用 < 和 > 符号来进行输入/输出的重定向，通常是表示将右侧/左侧的内容作为程序的输入/输出 

    ./test < file               # 将 file 内容作为标准输入
./test << 标识符             # 从标准输入中读取内容直到遇到标识符
./test > file               # 以覆盖方式输出到正确信息到 file
./test >> file              # 以追加方式输出到正确信息到 file
./test < file1 > file2      # 将 file1 作为 test 程序的输入，结果输出到 file2

    管道

    管道操作符 | 用于临时存储数据，它仅能处理经由前面一个指令传出的正确输出信息，错误信息会被忽略；与输入/输出重定向不同的是它能将多个指令连接起来执行，而不是单纯地操作一个指令的输入/输出 。例如：

    # 将 test 程序的输出作为 main 程序的输入
./test | ./main

    可以通过自定义管道来建立不同命令之间的连接

    mkfifo name     # 创建管道
rm name         # 删除管道

    我们把管道当做一个临时储存即可，例如

    mkfifo pipe     # 创建管道 pipe
ls > pipe       # 命令输入到 pipe 中
car < pipe      # 命令从 pipe 中输出， pipe 被清空

    需要注意管道运算的效率很低，在大规模数据处理中应该减少使用。

    标准流

    • 0 - 标准输入流 (stdin)
    • 1 - 标准输出流 (stdout)
    • 2 - 标准错误流 (stderr)
    • & 表示等同于的意思

    我们将标准错误流导向垃圾空间，从而不输出错误信息

    2 > /dev/null    # 将错误扔进垃圾空间  
2 > &1           # 将 2 的输出重定向与 1 相同

    后台运行

    使用 nohup 命令让命令在后台执行

    nohup ping baidu.com & 1 > /dev/null

    常用命令

    我们先总结一些简单的执行命令：

    命令 作用
    cat 用于查看文件内容，可以输出到另一个文件中，或是将文件内容输出到控制台
    wget 用于下载网络数据，没有参数时表示下载单个文件
    touch 用于创建文件（如果已经存在，则会修改文件的时间）
    tree 查看目录结构
    ps 显示进程信息
    eval 用于执行多个命令
    exec 不创建子进程，执行后面的命令，执行完毕后自动 exit
    time 用于统计后面命令的执行时间
    find 查询命令
    type 查看命令类型
    exit 退出程序，可以带一个返回值

    read

    read [options] [variable-name]

    options：

    选项 作用
    -a 后跟一个变量，该变量会被认为是个数组，然后给其赋值，默认是以空格为分割符
    -p "提示信息" 在等待 read 输入时，输出提示信息
    -t time read 命令会一直等待用户输入，使用此选项可以指定等待时间
    -n number read 命令只接受指定的字符数，就会执行
    -s 隐藏输入的数据，适用于机密信息的输入
    -d 后面跟一个标志符，其实只有其后的第一个字符有用，作为结束的标志
    -e 在输入的时候可以使用命令补全功能

    变量名可以自定义，如果不指定变量名，会把输入保存入默认变量 REPLY ，之后需要通过该变量获取输入；如果只提供了一个变量名，则整个输入行赋予该变量；如果提供了一个以上的变量名，则输入行分为若干字，一个接一个地赋予各个变量，而命令行上的最后一个变量取得剩余的所有字，例如

    read -t 30 -p "Please input your name:" Name

    wc

    用来统计指定文件中的行数、单词数、字符数

    wc [options] file-lise

    options:

    选项 作用
    -l 统计行数
    -w 统计单词数
    -c 统计字节数，一个中文三字节
    -m 统计字符数，和 -c 不能同时出现

    curl

    用于向网页发起请求，没有参数时表示以 GET 方式获取网页

    curl src

    使用 -i 参数打印出服务器回应的 HTTP 标头；-s 参数将不输出错误和进度信息。

    cp

    cp file1 file2               # 复制 file1 为 file2
cp -r dir1 dir2              # 递归地复制，用于复制文件夹

    mv

    mv file dir                 # 移动文件到目录
mv folder dir               # 移动文件夹到目录

    rm

    rm file                     # 删除文件
rm -rf dir                  # 删除文件夹

    mkdir

    创建目录

    mkdir name                      # 创建目录
mkdir -p a/b/c                  # 创建多层不存在的目录

    pwd

    显示目录名

    pwd                         # 显示当前目录

    rmdir

    rmdir name                  # 移除目录

    cd

    进入目录

    cd /                        # 进入根目录
cd name                     # 进入 name 目录
cd ~                        # 进入上一次列出文件的目录
cd ..                       # 返回上一级目录
cd ../                      # 进入相对路径

    在 Windows 下切换磁盘目录的命令是

    E:/     # 进入 E 盘
D:/     # 进入 D 盘
F:/     # 进入 F 盘

    ls

    列出目录下文件

    ls name                     # 列出目录下所有文件
ls -a                       # 列出所有文件包括隐藏文件
ls -lh                      # 列出所有文件详细信息
ls -l                       # 列出所有文件和信息

    其中 ls -l 命令等价于 ll 命令。

    tar

    用于解压/压缩文件

    .tar                        # 用 tar -xvf 解压
.gz                         # 用 gzip -d 解压
.tar.gz/.tgz                # 用 tar -zxvf 解压
.rar                        # 用 unrar e 解压
.zip                        # 用 unzip 解压

    压缩命令

    tar -zcvf file.tar.gz /file     # 压缩 file 为 .tar.gz
zip -q -r file.zip file/        # 压缩 file 为 .zip

    help

    在不清楚命令的用法时，使用

    help command_name
command_name --help

    来获得命令的详细说明。这两种形式都可以尝试，不同的命令支持不同的形式。

    man

    命令手册，有一些命令支持这种查询方式

    man command_name

    命令类型

    内置命令时系统启动时就载入内存、常驻内存的命令，执行效率高，但是占用资源，例如 cd 命令；外置命令时系统从硬盘中读取，然后载入内存的命令。我们可以查看命令的类型

    $ type cd
cd is a shell builtin

    变量

    命名规则

    变量名只能包含数字、字母和下划线，并且不能以数字开头，区分大小写。Bash 将所有变量视为字符串。

    定义变量

    定义变量时，变量名不加 $ ，并且变量名和等号之间不能有空格

    variableName="value"

    要使用一个定义过的变量，只要在变量名前面加 $ 即可

    your_name="mozhiyan"
echo $your_name
echo ${your_name}

    变量名外面的花括号是可选的，加花括号是为了帮助解释器识别变量的边界，推荐给所有变量加上花括号，这是个好的编程习惯。

    还可以定义只读的变量

    readonly var="hello"

    修改变量

    如果需要改变变量，可以利用变量叠加

    test=123
${test}                 # 取值
${#test}                # 变量长度
${test:start}           # 获得 start 位置后面的字符
${test:start:length}    # 获得 start 位置后面的 length 个字符
${test#word}            # 从开头删除最短的匹配 word 的部分
${test##word}           # 从开头删除最的匹配 word 的部分
${test%word}            # 从结尾删除最短的匹配 word 的部分
${test%%word}           # 从结尾删除最长的匹配 word 的部分
${test/pattern/string}  # 用 string 代替第一个匹配的 pattern
${test//pattern/string} # 用 string 代替所有匹配的 pattern

    这里 word 中可以使用通配符。

    扩展变量

    在使用变量时，经常要考虑变量是否为空的问题。这时就可以利用扩展变量

    ${test:-word}   # 如果 test 为空，返回 word
${test:=word}   # 如果 test 为空，则 test=word，返回 word
${test:?word}   # 如果 test 为空，word 作为 stderr 的输出，否则输出 test 值（用于返回错误信息）
${test:+word}   # 如果 test 不为空，返回 word，否则什么也不做

    例如

    find -name "*.tar.gz" -type f -mtime +7 | xargs rm -f

    其中参数

    • -name "*.tar.gz" 指定在名称模式下匹配 .tar.gz 结尾的文件
    • -type f 指定查找普通文件，f 代表普通文件
    • -mtime +7 查找最后修改时间超过 7 天的文件
    • xargs 接收来自标准输入的数据并将其转换为下一个命令的输入

    这条命令的作用是：查找当前目录及其子目录下所有最后修改时间超过7天的.tar.gz文件，并将这些文件删除。这个命令在需要清理旧的压缩备份文件时非常有用。

    unset variable_name

    shell 中含有一些特殊变量

    变量 含义
    $0 当前脚本的文件名
    $n 传递给脚本或函数的参数。n 是一个数字，表示第几个参数
    $# 传递给脚本或函数的参数个数

    $@ 传递给脚本或函数的所有参数。被双引号包含时，与 $* 稍有不同
    $? 上个命令的退出状态，或函数的返回值。返回 0 表示执行成功，否则失败

    基本运算

    条件运算

    条件测试的常用语法为

    语法 说明
    test expression 利用 test 命令进行测试，可以指定参数，最为高级
    [ expression ] 利用 [] 测试，注意两边各有一个空格
    [ expression ]( expression ) 一种更新的语法格式，同时检测文件是否有权限
    ((expression)) 用于 if 语句的条件测试，两边不需要空格

    需要注意条件表达式的值不会返回，只能通过 $? 获取错误码。

    [ $a -a $b ]

# 获得之前的返回值
var=$?

    test

    利用参数可以进行多重条件判断

    $ test -r file -a -x file

    其中 -a 表示 and 运算符，因此上面代码在 file 同时具有 -r,-x 权限时为真。类似还有

    $ test -r file -o -x file
$ test ! -r file

    其中 -o,! 分别为“或”和“非”运算符。

    双中括号

    在 test 和 [] 语法中，需要使用 -a,-o,!,=,!= 等进行逻辑判断，并且只支持比较简单的运算

    test $a -a $b
[ $a -o $b ]

    而双中括号是更现代的语法，可以直接使用 &&,||,==,!= 判断

    [ $a && $b ]( $a && $b )

    并且 ==,!= 支持模式匹配，还有 =~ 算符支持正则表达式

    [[ $sum =~ [1-3] ]]

    要求 sum 的值必须是 1,2,3 之一才为真。

    另外，在引用变量时，直接使用 [] 可能导致出现错误的变量引用

    [ "test test" -o "test test" ]
[ "test test" || "test test" ]( "test test" || "test test" )

    关系运算

    关系运算符只支持数字，不支持字符串，除非字符串的值是数字

    运算符 作用
    -eq [equal] 检测两个数是否相等，相等返回 true
    -ne [not equal] 检测两个数是否相等，不相等返回 true
    -gt [greater than] 检测左边的数是否大于右边的，如果是，则返回 true
    -lt [less than] 检测左边的数是否小于右边的，如果是，则返回 true
    -ge [greater than or equal] 检测左边的数是否大于等于右边的，如果是，则返回 true
    -le [less than or equal] 检测左边的数是否小于等于右边的，如果是，则返回 true

    例如比较 a 和 b 是否相等：

    [ $a -eq $b ]

    条件表达式要放在方括号之间，并且要有空格。

    算数运算

    原生 bash 不支持简单的数学运算，但是可以通过 awk 和 expr 实现。使用 exper 进行运算：

    val=`expr 2 + 2`

    注意相加的数之间要有空格。

    常见的算术命令有

    命令 作用
    (()) 高效整数算符
    let 整数算符，但是 (()) 效率更高
    expr 可以用于整数运算，但还有更多功能
    bc Linux 下的计算器，适合整数和小数运算
    $[] 用于整数运算
    awk 可用于整数和小数运算
    declare 定义变量值和属性，-i 参数可用于定义整型变量运算

    双小括号

    操作方法如下

    ((i=i+1))
i=$((i+1))
((8>7&&5==5))
echo $((2+1))

    利用它的计算特性，可以实现一个简单的运算器

    #! /bin/bash
# calculator.sh

echo $(($1))

    然后调用

    bash calculator.sh "1 + 2"

    这样参数传入后直接由 (()) 进行计算，然后取值输出。

    expr

    expr 命令根据空格区分传入参数，因此需要注意在某些需要传入元字符的情况下可能不好用。例如

    expr 5 * 3  # 报错
expr 5 \* 3 # 正确
expr 5 \> 3 # 正确

    还可以计算传入参数的长度，例如

    expr length 1234567*()  # 第 2 个参数长度为 10

    expr 还支持模式匹配。首先给出 : 参数表示启动匹配，然后附加正则表达式进行匹配。例如

    expr hello.jpg ":" ".*\.jpg"

    这里 .*\.jpg 就是正则匹配以 .jpg 结尾的字符串，最后返回匹配字符串的长度。

    awk

    awk 可以利用 print 函数计算，例如

    $ echo "3.2 1.2" | awk '{print $1+$2}'

    通过命令形式自动完成计算。

    字符串运算

    运算符 作用
    = 检测两个字符串是否相等，相等返回 true
    != 检测两个字符串是否相等，不相等返回 true
    -z 检测字符串长度是否为 0，为 0 返回 true
    -n 检测字符串长度是否为 0，不为 0 返回 true

    由于表达式不返回特定的值，只能直接用于判断，我们给出一个例子

    $ [ -n "test" ] && echo "full" || echo "empty"

    这就是一个简易的条件分支。

    递推列表

    我们经常需要生成等差数列之类的操作，这可以通过大括号扩展和 seq 命令实现。

    大括号扩展

    基本语法为

    {start..end}
{start..end..step}

    这种方式生成递增的数列。同时还可以对字母递推，例如

    {a..e}                      # a b c d e
{file1,file2,file3}.txt     # file1.txt file2.txt file3.txt

    使用 seq 命令指定生成范围

    seq 10 100
var=$((`seq 10 100`))

    利用此命令，我们可以将生成数值直接计算。例如

    这里 " + " 用于增加空格，这样传给 expr 计算才不会出错。

    正则表达式

    正则表达式是在一个或多个文件内容中搜索某一特定字符模式。Linux 中所有命令的输出结果都被分割符（默认为空格）划分为几部分， $1 、 $2 依次表示这些部分， $0 表示整个命令， $NF 表示一行有几部分，NF 表示一行的最后一个部分，NR 表示第几行。

    通配符

    首先要了解匹配模式中的通配符

    通配符 作用
    c* 匹配 0 个（即空白）获取多个字符 c
    . 匹配任何一个字符
    [xyz] 匹配方括号（集合）中的任意一个字符
    [^xyz] 匹配不包括方括号中的字符的全部字符
    ^ 匹配行的开头
    $ 匹配行的结尾

    如果要匹配通配符，需要加反斜杠 \ 转译。

    grep

    grep 具有如下格式：

    grep [options] method file

    options:

    选项 作用
    -c 列出包含模式的行数
    -i 忽略模式中的字母大小写
    -l 列出带有匹配行的文件名
    -n 在每行的最前面列出行号
    -v 列出没有匹配模式的行
    -w 把表达式作为一个完整的单字来搜索

    除 -w 选项外其它选项均可在 egrep 和 fgrep 命令中使用。例如：

    grep -c 'main' main.cpp

    在 main.cpp 文件中匹配 main 字串，同时列出包含模式的行数。

    变种

    grep 有两个变种，其中 egrep 在调用时自动使用了 -F 选项，语法格式与 grep 相同，多用于在一个或多个文件内容中利用扩展的正则表达式的元字符搜索特定的模式

    元字符 作用
    + 匹配一个或多个前导字符
    a b 匹配 a 或 b
    (RE) 匹配括号中的正则表达式 RE

    fgrep 在调用时自动使用了 -E 选项，搜索速度比 grep 快，可以一次搜索多个模式，不能搜索正则表达式，将通配符当作普通字符处理。

    单行模式

    使用 (?s) 前缀修饰的表达式在单行模式下匹配，即 . 可以匹配包括 \n 换行符在内的任何字符。

    sed

    用于搜索和替换字符串，具有如下格式：

    sed [options] 'expression' file

    其中 options 主要有：

    • -i 修改后文件替换原文件
    • -n 不输出原文件，只输出命令执行后的结果
    • -f 使用命令文件，即从文件中读取命令

    expression 具有格式：

    [范围]s/正则表达式/替换模式/[操作标识]

    可用的操作标识包括

    • g 表示要替代每一行中出现的全部模式
    • n 只替代前 n 行出现的模式
    • d 删除匹配模式

    可以同时使用两个命令表达式，但它们之间要用 ; 隔开，例如

    '/hello/d;s/abc/ABC/g'

    这就是两个匹配模式； sed 会返回操作的结果，因此可以将结果导入一个文件

    sed '/hello/d;s/abc/ABC/g' file1 > file2

    当然，也可以选择原地修改文件，只需要使用 -i 参数

    sed -i '/hello/d;s/abc/ABC/g' file

    注意替换模式中的 & 表示将匹配内容放在这个位置，如果需要替换为 &，应该在前面加上 \ 转义。

    awk

    awk 比较特殊，它可以使用一些语法命令，具有格式：

    awk '{commands}'

    其中 commands 为一个或多个命令（常用 print ）；命令表达式之间用 ; 分隔

    • -f 将从该标志之后的文件中读取指令而不是从命令行读取（即可以将指令存在文件中读取）
    • -Fc 表示字段之间分隔符是 c 而不是默认空白字符（c 是自定义的字符）

    BEGIN 和 END

    命令中较为重要的是开始 BEGIN 和结束 END 命令，具有格式： 

    awk 'BEGIN {...} END {...}'

    BEGIN：之后列出的操作（在 {} 内）将在 awk 开始扫描输入之前执行，即在获得文件输入前进行输出，使用 BEGIN 来显示变量和预置（初始化）变量。

    END：之后列出的操作将在扫描完全部的输入之后执行，使用 END 来输出最终结果。

    for 循环

    我们用一个例子说明 for 循环的使用。在 command 文件中存放如下指令： 

    {count[length($1)]++} END {for (i=1;i<9;i++) print "There are " count[i] "users"}

    则可以在控制台中读取 command 中的指令

    awk -F: -f command < test

    这表示用 : 作为字段分隔符（这样就会忽略空格），从之后的 command 文件中读取命令

    条件语句

    同样的，我们用一个例子说明 if 的使用：

    awk -F: '{if (length($1) == 3) print $0}' test

    这个命令是说如果传入参数长度是 3 ，就输出脚本名。

    自建函数

    shell 提供了一些简单的函数可供 awk 使用。

    index

    查找字符串的函数 index ，它返回子字符串第一次出现的位置，若没有返回 0

    'index(str, substr)'

    返回字符串长度的函数 length

    'length(str)'

    在源字符串中按正则表达式匹配子字符串，若有多条符合，则返回最左方的子字符串

    'match(str, 正则表达式)'

    同时内建变量

    RSTART：符合条件的子字符串在原字符串中的位置，若没有则为 0

    RLENGTH：符合条件的字符串长度，若没有则为 -1

    最终会返回 RSTART的值。

    截取字符串的函数 substr

    'substr($row, start, length)'

    获取 row 变量中以 start 开头，长度为 length 的部分。例如截取第二列（78944）中的44

    echo "123456 78944" | awk '{print substr($2, 4, 2)}'
